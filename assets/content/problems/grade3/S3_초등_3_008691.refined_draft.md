[문제 개요]
- 문제 번호는 `80.`으로 보인다.
- 문제 문장은 `원의 반지름과 지름을 나타내는 선분을 각각 찾아 보세요.`로 보이며, 끝부분 `보세요.`는 줄바꿈되어 아래줄에 이어진다.
- 가운데에는 원 도형이 크게 배치되어 있고, 원 안에 여러 선분과 라벨이 들어 있다.
- 도형 아래에는 선택 문장 `원의 반지름 (ㄱ, ㄴ, ㄷ), 원의 지름 (ㄱ, ㄴ, ㄷ)`가 인쇄되어 있다.
- 하단에는 `(정답)`과 `(해설)`이 함께 인쇄되어 있는 상태로 보인다.

[원본에 보이는 요소]
- 왼쪽 위:
  - 작은 빈 네모 표시처럼 보이는 기호
  - `80.`
  - 문제 문장
- 중앙 오른쪽:
  - 큰 원 1개
  - 원 내부의 중심점처럼 보이는 분홍색 점 1개
  - 중심 근처의 문자 `O`
  - 검은 실선 여러 개
  - 파란 점선/파란 곡선 보조선 여러 개
  - 원 내부에 원형 작은 라벨 3개:
    - `ㄱ`
    - `ㄴ`
    - `ㄷ`
- 원 아래:
  - `원의 반지름 (ㄱ, ㄴ, ㄷ), 원의 지름 (ㄱ, ㄴ, ㄷ)`
- 하단:
  - `(정답)ㄷ, ㄴ`
  - `(해설)원의 반지름 ㄷ, 원의 지름 ㄴ`

[수식/관계 구조]
- 등호가 있는 수식은 보이지 않는다.
- 계산식이나 식 변형도 보이지 않는다.
- 대신 도형의 관계 구조가 보인다.
- 보이는 관계를 문자로 정리하면:
  - 원 중심 근처의 점 `O`
  - `O`에서 원 둘레까지 이어지는 선분들 중 일부는 반지름 후보로 보임
  - 원을 양 끝점으로 가로지르거나 중심을 지나가는 선분은 지름 후보로 보임
- 원 내부에는 선분이 서로 겹치고 교차하며, 각 선분에 `ㄱ`, `ㄴ`, `ㄷ` 라벨이 붙어 있다.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄 변형이나 계산식 변환이 없다.

[빈칸 정책]
- 이 문항은 계산 빈칸이 아니라 선택형 도형 문제로 보인다.
- 따라서 추론으로 빈칸 값을 채우지 않는다.
- 원본에 인쇄된 `(정답)`, `(해설)`은 관찰 사실로만 기록하고, 학습자가 채워야 할 입력값으로 간주하지 않는다.
- `ㄱ`, `ㄴ`, `ㄷ`의 정확한 대응은 이미지에서 읽히는 대로만 기록하고, 불명확한 경우는 TODO로 남긴다.

[반복 패턴]
- 원 중심에서 둘레로 향하는 선분은 반지름 후보라는 반복 패턴으로 보인다.
- 원을 중심을 지나 반대편 둘레까지 관통하는 선분은 지름 후보라는 반복 패턴으로 보인다.
- 파란 점선 곡선은 실제 답 선분이 아니라 강조/안내용 보조선처럼 반복적으로 사용된다.
- 선분 옆의 작은 원형 라벨 `ㄱ`, `ㄴ`, `ㄷ`는 각 선분을 구분하는 반복 마커이다.

[DSL 작성 힌트]
- 원 1개를 중심 도형으로 두고, 내부의 선분을 개별 객체로 분리하는 것이 좋다.
- 중심점 `O`와 분홍색 점은 서로 다른 요소로 분리해서 취급하는 것이 안전하다.
- 검은 실선:
  - 위쪽에서 중심/둘레로 이어지는 대각선
  - 중심에서 오른쪽 둘레로 이어지는 수평선
  - 중심에서 아래 둘레로 이어지는 수직선
  - 왼쪽 위에서 중심 쪽으로 내려오는 대각선
- 파란 점선:
  - 위쪽에서 왼쪽으로 휘는 곡선
  - 중심에서 아래로 휘는 곡선
  - 중심에서 오른쪽으로 휘는 곡선
- 라벨 `ㄱ`, `ㄴ`, `ㄷ`는 각 선분 옆에 작은 원형 태그 또는 텍스트로 배치한다.
- 문제 문장은 상단 좌측, 도형은 우측 중앙, 선택 문장은 도형 아래, 정답/해설은 하단에 별도 텍스트 블록으로 배치하는 구성이 적절하다.
- 원 내부 선분은 교차와 겹침이 있으므로, 끝점 기준으로 분리해서 그리면 레이아웃 재현이 쉽다.

[semantic/solvable 힌트]
- 이 문제는 반지름과 지름의 정의를 보고 도형에서 해당하는 선분을 고르는 유형이다.
- 중심을 지나 원 둘레까지 가는 선분은 반지름으로 해석된다.
- 중심을 지나 원의 양쪽 둘레를 잇는 선분은 지름으로 해석된다.
- 원본 하단에는 다음이 인쇄되어 보인다:
  - `(정답) ㄷ, ㄴ`
  - `(해설) 원의 반지름 ㄷ, 원의 지름 ㄴ`
- 다만 이는 원본에 보이는 인쇄 정보로만 취급하고, 풀이 과정을 새로 전개하지 않는다.

[주의 및 TODO]
- `ㄱ`, `ㄴ`, `ㄷ`의 정확한 선분 대응은 해상도 영향으로 다소 흐릴 수 있다.
- 원 안의 `O`와 분홍색 점은 각각 별도 요소로 인식해야 한다.
- 파란 점선의 정확한 시작점/끝점은 일부 구간이 겹쳐 보여 애매하다.
- 문제 문장의 줄바꿈은 실제 편집 줄바꿈인지 자동 줄바꿈인지 확정하기 어렵다.
- 정답과 해설이 하단에 인쇄되어 있지만, 이를 입력칸으로 착각하지 않도록 주의한다.
- 본 문항에는 `a/b ÷ n -> a/(b×n)` 형태의 변환이 보이지 않는다.
