[문제 개요]
- 초등 수학 선택형 시각 문제이다.
- 문제 번호는 `97.`로 보인다.
- 지시문은 `컴퍼스를 2 cm가 되도록 벌린 것을 찾아 선택하세요.`이다.
- 위쪽에 후보 그림 3개가 가로로 배치되어 있고, 아래쪽 왼편에 `(정답)` 표시가 붙은 그림 1개가 따로 있다.
- 화면 하단에는 `(해설)`로 시작하는 문장이 1줄로 인쇄되어 있다.

[원본에 보이는 요소]
- 보이는 텍스트
  - `97.`
  - `컴퍼스를 2 cm가 되도록 벌린 것을 찾아 선택하세요.`
  - `(정답)`
  - `(해설)`
  - `컴퍼스의 침을 자의 눈금 0에 맞추고 연필심을 자의 눈금 2에 오도록 컴퍼스를 벌린 것을 찾습니다.`
- 그림 구성
  - 컴퍼스와 자가 함께 그려진 후보 그림 3개
  - 아래쪽에 별도 그림 1개
- 시각적 공통 요소
  - 상단에 컴퍼스 본체와 두 다리
  - 왼쪽 다리 끝의 침
  - 오른쪽 다리 쪽의 연필심
  - 하단에 자
- 자의 눈금 표시
  - `0, 1, 2, 3` 숫자가 크게 보인다.
  - 짧은 눈금이 촘촘히 반복된다.
- 색/형태
  - 자는 하늘색 계열
  - 컴퍼스는 회색/청록 계열 선
  - 연필은 노란색 몸통과 연두색 상단이 있는 세로 막대처럼 보인다

[수식/관계 구조]
- 이 문항은 등호가 이어지는 수식 변형 문제가 아니라, 그림의 위치 관계를 보고 고르는 유형이다.
- 핵심 관계는 다음과 같이 읽힌다.
  - 컴퍼스의 침 위치 ↔ 자의 `0`
  - 컴퍼스의 연필심 위치 ↔ 자의 `2`
- 따라서 의미상으로는 “컴퍼스 벌림이 2 cm에 해당하도록 보이는 그림”을 찾는 구조이다.
- 다만 여기서는 답을 계산하거나 특정 후보를 단정하지 않고, 보이는 관계만 기록한다.

[등호별 변환 의미]
- 해당 없음.
- 그림 비교형 문제이므로 등호 연쇄 변형은 보이지 않는다.

[빈칸 정책]
- 이 화면에는 별도의 수학 빈칸이 보이지 않는다.
- 정답을 추론해서 채워 넣지 않는다.
- `(정답)` 표시는 답안을 입력하는 칸이 아니라, 아래쪽 별도 그림에 붙은 라벨로만 취급한다.
- 만약 DSL에서 선택지나 라벨 자리가 필요하면, 추론값 대신 보이는 텍스트와 그림 배치만 유지한다.

[반복 패턴]
- 같은 유형의 컴퍼스+자 그림이 반복된다.
- 반복되는 구조
  - 상단의 컴퍼스
  - 하단의 자
  - 자 위에서 침과 연필심이 서로 다른 눈금에 놓임
- 후보 3개는 위쪽에 가로로 정렬되어 있고, 크기와 기본 구성이 비슷하다.
- 아래쪽의 `(정답)` 그림은 위 후보들과 같은 도식 구조이지만, 왼쪽에 라벨이 추가되어 별도 블록처럼 보인다.
- 자의 짧은 눈금은 균일 간격의 반복 패턴으로 처리하기 적합하다.

[DSL 작성 힌트]
- 큰 영역 분할
  - `문제문장`
  - `후보그림들`
  - `정답라벨_별도그림`
  - `해설문장`
- 배치 힌트
  - 후보 3개는 상단 한 줄에 좌→우로 정렬
  - 아래쪽 왼편에 정답 라벨이 붙은 별도 그림 배치
  - 해설 문장은 맨 아래에 한 줄 텍스트로 배치
- 그림 내부 분해 요소
  - 상단 컴퍼스 본체
  - 왼쪽 다리의 침
  - 오른쪽 다리의 연필심
  - 하단 자
  - 자의 숫자 `0, 1, 2, 3`
  - 자의 짧은 눈금 반복
- 렌더링/레이아웃 힌트
  - 자 숫자는 크게 읽히도록 배치
  - 짧은 눈금은 동일 간격 반복으로 표현
  - 침과 연필심의 위치는 자 눈금과의 상대 위치가 중요하므로, 눈금 기준 정렬이 핵심
  - `(정답)` 텍스트는 그림 외곽의 라벨로 따로 처리
- 텍스트 처리
  - 문제문장과 해설문장은 그대로 별도 텍스트 블록으로 두는 편이 좋다.
  - 해설문장을 풀이 결과로 재작성하지 말고 원문 그대로 유지한다.

[semantic/solvable 힌트]
- 보이는 해설 문장에 따르면, 의미상 기준은 `침을 0에`, `연필심을 2에` 맞추는 것이다.
- 따라서 이 문항은 컴퍼스 벌림을 눈금 길이로 비교하는 시각 문제로 해석된다.
- 그러나 후보들 각각이 정확히 어떤 값을 나타내는지, 또는 어떤 그림이 정답인지 이 메모에서는 단정하지 않는다.
- 정답이 이미지에 따로 표시되어 있어도, DSL에는 “정답 그림이 별도로 존재한다”는 구조만 반영한다.
