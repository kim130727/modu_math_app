[문제 개요]
- 객관식 5지선다 문제이다.
- 지문: **“반지름이 1 cm인 원을 그리는 순서를 바르게 나타낸 것을 고르세요.”**
- 화면 상단에 문제 번호 **12.**가 보인다.
- 중앙에는 연한 하늘색 테두리의 큰 그림 박스가 있고, 그 안에 원을 그리는 과정을 나타내는 3개의 장면이 가로로 배치되어 있다.
- 하단에는 선택지 ①~⑤가 있고, 그 아래에 **(정답) ②**와 **(해설)** 문장이 보인다.

[원본에 보이는 요소]
- 문제 번호: **12.**
- 지문 텍스트: **반지름이 1 cm인 원을 그리는 순서를 바르게 나타낸 것을 고르세요.**
- 중앙 그림 박스: 연한 하늘색/민트색 테두리의 직사각형
- 그림 박스 안의 단계 표기: **ㄱ, ㄴ, ㄷ**가 각각 원형 라벨 안에 들어가 있음
- 3개의 장면이 왼쪽에서 오른쪽으로 배치됨
- 선택지:
  - **① ㄱ, ㄴ, ㄷ**
  - **② ㄴ, ㄷ, ㄱ**
  - **③ ㄷ, ㄱ, ㄴ**
  - **④ ㄱ, ㄷ, ㄴ**
  - **⑤ ㄴ, ㄱ, ㄷ**
- 하단 정답 표기: **(정답) ②**
- 하단 해설:
  - **(해설) ㄴ 원의 중심 정하기 → ㄷ 반지름만큼 컴퍼스 벌리기 → ㄱ 컴퍼스의 침을 원의 중심에 꽂고 원 그리기**

[수식/관계 구조]
- 이 문항은 수식 계산 문제가 아니라, **그림 3장의 순서를 배열하는 관계 구조**이다.
- 관계는 다음과 같이 보인다:
  - **ㄴ**: 원의 중심 정하기
  - **ㄷ**: 반지름만큼 컴퍼스 벌리기
  - **ㄱ**: 컴퍼스의 침을 원의 중심에 꽂고 원 그리기
- 선택지들은 이 3개 라벨의 순서만 다르게 배열되어 있다.
- 등호나 수식 변형은 보이지 않는다.

[등호별 변환 의미]
- 해당 없음.
- 화면에 등호(=)를 사용한 변형 연쇄가 보이지 않는다.
- 따라서 등호별 단계 설명은 필요하지 않다.

[빈칸 정책]
- 빈칸 형태의 입력칸은 보이지 않는다.
- 다만 DSL 작성 시에는 **추론으로 누락 요소를 채우지 말고**, 화면에 보이는 텍스트와 도형만 반영해야 한다.
- 정답이 화면에 **인쇄되어 보이므로** 관찰 사실로는 기록 가능하지만, 보이지 않는 빈칸이 있다면 임의로 채우지 않는다.
- 이 문제에서는 정답 표기 **(정답) ②**가 보이지만, 풀이 결과를 새로 계산해서 적는 방식은 피한다.

[반복 패턴]
- 단계 그림이 3개 반복된다.
- 각 단계 위에는 원형 라벨이 하나씩 붙고, 라벨은 **ㄱ / ㄴ / ㄷ** 중 하나이다.
- 보기도 같은 세 라벨의 순열이 반복되는 형태이다.
- 시각적으로는 “장면 3개 + 라벨 3개 + 순서 선택지 5개”라는 패턴이다.
- 하단 해설은 화살표(→)로 단계 흐름을 연결한 문장 형태로 반복 구조를 보강한다.

[DSL 작성 힌트]
- 전체 구조:
  - 상단: 문제 번호 + 지문
  - 중앙: 연한 색 테두리의 큰 그림 박스
  - 하단: 선택지 5개(2줄 배치) + 정답 표기 + 해설 텍스트
- 그림 박스 내부:
  - 3개의 소그림을 **좌→중→우** 순서로 배치
  - 각 소그림 위에 **동그라미 안 한글 자모** 라벨 배치
- 각 소그림의 요소:
  - 왼쪽: 큰 원 도형, 내부에 점/작은 원이 함께 보임
  - 가운데: 작은 점과 작은 원 표시
  - 오른쪽: 컴퍼스와 자(눈금자) 형태의 도구
- 도구 표현은 세밀한 실제 묘사보다 **도형적 상징**으로 단순화해도 된다.
- 선택지 영역:
  - ①~⑤ 번호와 조합 텍스트를 두 줄로 배치
  - 줄바꿈과 간격을 원본처럼 유지
- 하단 텍스트:
  - **(정답) ②**
  - **(해설) ㄴ 원의 중심 정하기 → ㄷ 반지름만큼 컴퍼스 벌리기 → ㄱ 컴퍼스의 침을 원의 중심에 꽂고 원 그리기**
- 렌더링 힌트:
  - 그림 박스는 연한 하늘색 테두리 직사각형
  - 라벨은 작은 원 안에 글자 1개씩
  - 오른쪽 도구는 컴퍼스 다리와 자 눈금이 보이도록 배치
  - 전체는 좌우 여백이 넓은 시험지 스타일

[semantic/solvable 힌트]
- 이 문항의 의미는 “원 그리기 절차의 올바른 순서”를 묻는 것이다.
- 그림과 해설이 함께 존재하므로, 장면의 의미는 다음과 같이 읽힌다:
  - 먼저 중심을 정하고
  - 다음에 반지름만큼 컴퍼스를 벌리고
  - 마지막으로 컴퍼스로 원을 그린다
- 단, 이 항목은 **문항의 의미 설명**일 뿐이며, 추론 결과를 새로 계산하는 용도로 쓰지 않는다.
- 화면에 **(정답) ②**가 보이므로, 정답 표기가 존재한다는 사실은 기록 가능하다.

[주의 및 TODO]
- 원본 이미지 기준으로 라벨 **ㄱ/ㄴ/ㄷ**의 실제 위치와 장면 대응을 최종 확인해야 한다.
- 왼쪽 그림의 원 내부 점과 작은 원의 정확한 도식적 의미는 화면만으로는 단정하지 말고, 필요 시 TODO로 남긴다.
- 오른쪽 그림의 자 눈금, 컴퍼스 다리 각도, 접점 위치는 작은 요소이므로 DSL에서는 단순화 가능하지만, 위치 관계는 유지해야 한다.
- 해설 문장에 있는 화살표 연결은 텍스트 그대로 반영하되, 그림과 해설을 섞어 쓰지 않는다.
- 빈칸을 추론으로 채우지 말 것.
- 등호 변형은 없음.
- a/b ÷ n 형태의 변환도 없음, 따라서  
  **"나누기 n은 분모의 ×n으로 반영된다."**  
  문장은 이 문항에는 해당되지 않는다.
