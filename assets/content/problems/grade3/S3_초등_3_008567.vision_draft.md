[전체적인 구성]
- 한 문제짜리 선택형 수학 문항입니다.
- 화면 왼쪽 위에 문제 번호와 지시문이 있고, 가운데에는 연두색 테두리의 선택지 상자가 있습니다.
- 오른쪽 아래 쪽에는 선택지를 적는 괄호 표기가 따로 보입니다.
- 하단 왼쪽에는 `(정답)` 표시와 `(해설)` 표시가 있으며, 해설 문장이 두 줄로 제시되어 있습니다.
- 전체적으로 “같은 수를 다른 방식으로 나타내는 것”을 고르는 형식입니다.

[보이는 텍스트]
- `□ 32. 나타내는 수가 다른 하나를 찾아 기호를 선택하세요.`
- 선택지 상자 안:
  - `㉠ 214 + 214 + 214`
  - `㉡ 214 × 3`
  - `㉢ 214 + 3`
- 오른쪽에 따로:
  - `( ㉠ , ㉡ , ㉢ )`
- 아래쪽:
  - `(정답) ㉢`
  - `(해설) ㉠, ㉡ 214 + 214 + 214 = 214 × 3 = 642`
  - `㉢ 214 + 3 = 217`

[반복되는 시각 패턴]
- 선택지가 `㉠, ㉡, ㉢`처럼 동그라미 기호와 함께 세로로 나열되어 있습니다.
- 각 선택지는 모두 숫자 표현식 형태이며, 첫 두 선택지는 같은 수를 반복하거나 곱셈으로 나타낸 형태로 보입니다.
- 아래 해설에서도 `㉠`와 `㉡`가 한 묶음으로 제시되고, `=` 기호가 여러 번 이어진 등식 형태가 반복됩니다.
- `214 + 214 + 214 = 214 × 3 = 642`처럼 좌변에서 우변으로 단계적으로 바뀌는 표현이 보입니다.

[수식 단계]
- 해설에 등식이 연속으로 제시되어 있습니다.
- 첫 줄의 수식:
  - `214 + 214 + 214 = 214 × 3 = 642`
- 두 번째 줄의 수식:
  - `214 + 3 = 217`
- 각 줄은 서로 다른 선택지의 의미를 설명하는 형식으로 배치되어 있습니다.

[등호별 변환 의미]
- `214 + 214 + 214 = 214 × 3`
  - 보이는 형태상, 같은 수 `214`가 세 번 더해진 표현이 곱셈 표현으로 바뀐 것입니다.
- `214 × 3 = 642`
  - 곱셈식의 값이 계산 결과로 이어진 형태입니다.
- `214 + 3 = 217`
  - 덧셈식의 값이 계산 결과로 이어진 형태입니다.
- 다만 여기서는 계산 과정을 설명하는 것이 아니라, 해설에 적힌 등식 구조를 그대로 관찰한 것입니다.

[세부 내용]
- 문제 번호 `32`는 왼쪽 상단에 있습니다.
- 문제 문장은 한 줄로 길게 쓰여 있고, `다른`이라는 단어가 밑줄로 강조되어 있는 것처럼 보입니다.
- 선택지 상자는 연두색 테두리의 직사각형이며, 내부에 3개의 항목이 위에서 아래로 일정 간격으로 배치되어 있습니다.
- 각 항목은 기호 `㉠, ㉡, ㉢`와 수식이 같은 줄에 있습니다.
- 오른쪽 아래의 `( ㉠ , ㉡ , ㉢ )` 표기는 답을 적는 자리처럼 보이며, 선택 기호만 괄호 안에 나열되어 있습니다.
- `(정답)`은 작은 글씨로 왼쪽 아래에 있고, 그 옆에 `㉢`가 표시되어 있습니다.
- `(해설)` 아래에는 두 줄의 식이 있으며, 첫 줄은 `㉠, ㉡`에 대한 설명, 둘째 줄은 `㉢`에 대한 설명으로 보입니다.
- 식의 글자 크기는 문제 본문보다 해설 부분에서 조금 더 크게 느껴집니다.

[Python DSL 작성 힌트]
- 문제 본문과 선택지 영역, 답안 표시 영역, 해설 영역을 서로 다른 텍스트 블록으로 나누어 배치하는 것이 좋습니다.
- 선택지 박스는 사각형 테두리로 표현하고, 내부 항목은 세로 정렬로 놓는 구조가 적합합니다.
- 각 선택지 앞의 `㉠/㉡/㉢`는 원형 기호 텍스트로 처리하세요.
- 오른쪽 아래의 `( ㉠ , ㉡ , ㉢ )`는 별도 텍스트 객체로 두어, 선택지와 시각적으로 분리하는 것이 좋습니다.
- 해설의 등식은 한 줄에 길게 배치하거나, 필요하면 줄바꿈을 유지한 채 텍스트로 구현하세요.
- `=`가 연속으로 나타나는 수식은 한 줄 문자열로 보존해 등식 흐름이 드러나게 하는 편이 좋습니다.

[주의할 점]
- 정답을 새로 추론해서 적지 마세요. 화면에 보이는 `정답` 표기를 그대로 반영해야 합니다.
- 선택지의 의미를 풀이로 확장하지 말고, 보이는 수식 형태만 기록하세요.
- 해설의 수식은 계산 결과를 설명하는 텍스트일 뿐, 여기서 다시 계산 과정을 자세히 전개하지 마세요.
- `214 + 214 + 214`와 `214 × 3`의 관계를 설명하더라도, 문제 해결 서술로 바꾸지 마세요.
- 선택지 상자와 해설 영역은 공간적으로 떨어져 있으므로, DSL에서도 다른 구역으로 분리하는 것이 좋습니다.

[불확실한 부분]
- 문제 문장 앞의 작은 네모 `□`는 인쇄된 기호인지 체크박스인지 화면만으로 확정하기 어렵습니다.
- `다른`의 밑줄은 실제 밑줄인지, 강조 표기인지 약간 애매합니다.
- 선택지 상자 테두리의 정확한 색상은 연두색 계열로 보이지만, 화면 캡처 특성상 완전히 동일하게 단정하기는 어렵습니다.
- 해설 줄의 시작 위치와 들여쓰기는 이미지 해상도상 약간 오차가 있을 수 있습니다.
