[문제 개요]
- 문제 번호는 `52.`로 보이며, 왼쪽 위에 작은 네모 표시 `□`가 함께 있습니다.
- 지시문은 `알맞은 말을 선택해 보시오.`입니다.
- 중심이 `O`로 표시된 원 그림이 있고, 중심에서 원둘레로 향하는 선분 3개가 그려져 있습니다.
- 아래에는 문장형 선택 문항이 있고, 그 아래에 `(정답)`과 `(해설)`이 별도 텍스트로 제시되어 있습니다.
- 이 문제는 원의 기본 용어를 고르는 개념 확인형 문제로 보입니다.

[원본에 보이는 요소]
- 상단 왼쪽:
  - `□ 52. 알맞은 말을 선택해 보시오.`
- 중앙 도형:
  - 원 1개
  - 원의 중심 부근에 점과 문자 `O`
  - 중심 `O`에서 원둘레로 뻗은 선분 3개
  - 원둘레 쪽 점 표시는 `ㄱ`, `ㄴ`, `ㄷ`
- 문장:
  - `선분 ㄱO, 선분 ㄴO, 선분 ㄷO은 모두 원의 (지름, 반지름, 중심)입니다.`
- 정답 표기:
  - `(정답)반지름`
- 해설 표기:
  - `(해설)원의 중심과 원 위의 한 점을 잇는 선분은 모두 원의 반지름입니다. 한 원에서 원의 반지름의 수는 무수히 많습니다.`

[수식/관계 구조]
- 등호나 계산식은 보이지 않습니다.
- 관계 구조는 다음과 같이 읽힙니다.
  - `ㄱO`, `ㄴO`, `ㄷO`는 모두 중심 `O`에서 원둘레의 점으로 이어진 선분입니다.
  - 선택지 `(지름, 반지름, 중심)` 중 하나를 고르는 구조입니다.
- 보이는 사실과 수학적 의미를 분리하면:
  - 보이는 사실: 세 선분이 모두 원의 안쪽에서 시작해 원둘레까지 닿아 있습니다.
  - 추론되는 의미: 중심과 원 위의 점을 잇는 선분은 반지름입니다.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄 변형이나 식 변환은 보이지 않습니다.

[빈칸 정책]
- 이 자료에는 계산 빈칸이 보이지 않습니다.
- 선택지 괄호 `(지름, 반지름, 중심)`는 문장 안의 보기로 보이므로, DSL에서 재현할 때도 임의로 값을 채우지 않습니다.
- 화면에 보이는 정답 표기 `(정답)반지름`은 인쇄된 사실로만 기록할 수 있습니다.
- 빈칸이나 선택 결과를 추론해서 새로 작성하지 않습니다.

[반복 패턴]
- `선분 ㄱO, 선분 ㄴO, 선분 ㄷO`처럼 같은 구조의 선분 이름이 3번 반복됩니다.
- 세 선분은 모두 `중심 O → 원둘레의 점` 형태라는 공통 패턴을 가집니다.
- 도형에서는 중심에서 여러 방향으로 뻗는 방사형 구조가 반복적으로 드러납니다.

[DSL 작성 힌트]
- 도형 요소를 다음처럼 분리해 배치하는 것이 좋습니다.
  - 원 1개
  - 중심점 `O`
  - 중심에서 원둘레로 향하는 선분 3개
  - 원둘레 위의 점/표시 `ㄱ`, `ㄴ`, `ㄷ`
- 텍스트 영역은 최소 3블록으로 나누면 좋습니다.
  - 상단: 문제 번호와 지시문
  - 중단: 문장형 선택 문항
  - 하단: `(정답)`, `(해설)`
- 렌더링/레이아웃 힌트:
  - 도형은 페이지 상단 중앙보다 약간 오른쪽에 위치합니다.
  - `ㄱ`은 왼쪽 위 원둘레 근처
  - `ㄴ`은 아래쪽 원둘레 근처
  - `ㄷ`은 오른쪽 위 원둘레 근처
  - `O`는 원의 중심 근처에 작은 문자로 표시됩니다.
  - 선분은 가늘고 직선이며, 중심에서 바깥쪽 점까지 연결됩니다.
  - 선택지 문장은 줄바꿈되어 보일 수 있으므로, 한 줄로 고정하지 말고 자연스러운 줄바꿈을 허용하는 편이 좋습니다.
- 정답과 해설은 본문과 분리된 하단 텍스트 영역으로 처리하는 것이 적절합니다.
- 보기의 괄호 `(지름, 반지름, 중심)`는 텍스트 그대로 유지하는 것이 안전합니다.

[semantic/solvable 힌트]
- 이 문제는 원의 용어를 묻는 개념 확인 문제입니다.
- 보이는 선분들은 모두 중심 `O`와 원 위의 점을 잇는 형태로 해석됩니다.
- 해설 문장도 같은 의미를 직접 설명하고 있습니다.
- 다만, DSL 작성 시에는 정답을 새로 계산하거나 추론해 빈칸에 넣지 말고, 화면에 표시된 텍스트만 반영해야 합니다.
- 화면에 인쇄된 `(정답)반지름`은 관찰 사실로 다룰 수 있습니다.
