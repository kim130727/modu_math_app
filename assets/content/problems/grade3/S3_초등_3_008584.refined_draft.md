# [문제 개요]
- 문제 번호는 `51`로 보입니다.
- 지시문은 `묶이 다른 것을 선택하세요.`입니다.
- 화면에는 위쪽에 3개의 선택 그림이 가로로 배치되어 있고, 아래쪽 왼편에는 `(정답)` 표시와 함께 별도의 그림 1개가 있습니다.
- 더 아래에는 `(해설)` 문장과 계산식, 결론 문장이 이어집니다.
- 이 화면은 `선택지 3개 + 정답 표시 + 해설` 구조의 초등 수학 문제로 보입니다.

# [원본에 보이는 요소]
## 보이는 사실
- 좌측 상단에 작은 빈 사각형 기호처럼 보이는 요소가 있고, 그 옆에 `51.`이 있습니다.
- 지시문 텍스트는 `묶이 다른 것을 선택하세요.`로 읽힙니다.
- 위쪽 선택지는 3개이며 각각 안에 나눗셈 식이 1개씩 들어 있습니다.
  - `72 ÷ 3`
  - `84 ÷ 2`
  - `96 ÷ 4`
- 아래쪽 왼편에는 `(정답)` 라벨이 있고, 그 오른쪽에 또 하나의 같은 형태 그림이 있습니다.
- 정답 표시 옆 그림 안의 식은 `84 ÷ 2`로 보입니다.
- 해설은 `(해설)` 뒤에 계산식이 한 줄로 이어집니다.
- 해설 문장은 `따라서 묶이 다른 것은 84 ÷ 2입니다.`로 보입니다.

## 추론되는 수학적 의미
- 선택지들은 모두 나눗셈 식이며, 해설은 각 식의 계산 결과를 함께 적어 비교하는 형식으로 보입니다.
- 문제의 핵심은 `묶이 다른 것`을 고르는 분류 문제로 보이지만, 여기서는 의미를 단정하기보다 화면에 적힌 구조만 기록하는 것이 안전합니다.

# [수식/관계 구조]
## 보이는 수식
- 선택지 1: `72 ÷ 3`
- 선택지 2: `84 ÷ 2`
- 선택지 3: `96 ÷ 4`
- 정답 표시 옆 그림: `84 ÷ 2`
- 해설에 보이는 식:
  - `72 ÷ 3 = 24`
  - `84 ÷ 2 = 42`
  - `96 ÷ 4 = 24`

## 구조 관찰
- 각 선택지는 `수 ÷ 수` 형태의 단일 식입니다.
- 해설에서는 각 식의 오른쪽에 `=`와 결과값이 붙어 있습니다.
- 화면에는 긴 변형식 사슬은 없고, `식 = 값` 형태가 반복됩니다.

# [등호별 변환 의미]
- `72 ÷ 3 = 24`에서 `=`는 왼쪽 나눗셈식과 오른쪽 결과를 연결합니다.
- `84 ÷ 2 = 42`에서 `=`는 동일하게 식과 결과를 연결합니다.
- `96 ÷ 4 = 24`에서도 같은 역할입니다.
- 각 등호는 단계적 변형의 연쇄라기보다, 해설용으로 계산 결과를 나열한 형태로 보입니다.
- 화면상 등호 변형이 길게 이어지지 않으므로, 각 줄은 독립적인 `식 → 결과` 대응으로 이해하면 됩니다.

# [빈칸 정책]
- 화면에 보이는 빈칸은 보이지 않습니다.
- 만약 DSL 작성 과정에서 빈칸 슬롯이 필요해 보이더라도, 원본에 인쇄된 내용이 없는 부분은 추론값으로 채우지 말고 빈칸 상태로 유지해야 합니다.
- 정답이 화면에 이미 인쇄되어 보이더라도, 본 문서는 관찰 사실만 기록하고 별도의 빈칸 채우기는 하지 않습니다.
- 특히 선택지/정답 그림 안의 식은 보이는 문자열만 그대로 옮기고, 숨은 정답을 새로 계산해 넣지 않습니다.

# [반복 패턴]
## 보이는 반복
- 상단 3개 선택지는 모두 비슷한 연한 청록색 테두리의 사다리꼴/상자형 도형입니다.
- 각 도형 내부의 식은 가운데 정렬되어 있습니다.
- 도형 아래에는 작은 청록색 원형 요소가 2개씩 보이며, 바퀴처럼 보입니다.
- 3개의 선택지는 같은 스타일과 같은 크기로 반복됩니다.
- 하단의 정답 그림도 같은 도형 스타일을 사용하지만, 왼쪽에 `(정답)` 라벨이 붙어 구분됩니다.

## 레이아웃 반복
- 위쪽 3개 선택지는 수평으로 넓게 간격을 두고 배치됩니다.
- 해설은 `(해설)` 라벨 뒤에 식 나열이 한 줄로 이어집니다.
- 마지막 결론 문장도 한 줄 텍스트로 표시됩니다.

# [DSL 작성 힌트]
## 렌더링/레이아웃 힌트
- 상단 왼쪽에 문제 번호와 지시문을 먼저 배치하세요.
- 그 아래 상단 영역에 동일한 도형 컴포넌트 3개를 가로 정렬로 놓는 구성이 적합합니다.
- 각 도형은 외곽선, 내부 채움, 하단 원형 요소를 분리된 요소로 다루면 좋습니다.
- 도형 내부의 나눗셈 식은 가운데 정렬 텍스트로 표현하세요.
- 하단 왼쪽의 `(정답)` 라벨과 그림은 하나의 묶음으로 배치하는 것이 자연스럽습니다.
- 해설 영역은 `(해설)` 라벨, 계산식 한 줄, 결론 문장 순으로 세로 배치하면 원본과 유사합니다.
- 선택지와 정답 그림은 시각적으로 같은 종류의 도형이므로, 위치와 라벨 차이로만 구분하는 방식이 적합합니다.

## 구성 분해 힌트
- 문제 번호/지시문
- 선택지 도형 3개
- `(정답)` 라벨 + 정답 그림 1개
- `(해설)` 라벨 + 계산식 문자열
- 결론 문장

# [semantic/solvable 힌트]
- 이 문제는 나눗셈 식들을 비교하여 `묶이 다른 것`을 찾는 유형으로 보입니다.
- 해설에는 각 선택지의 계산 결과가 함께 적혀 있어, 비교 기준이 결과 또는 묶음의 성질과 관련되어 있음을 시사합니다.
- 다만 본 문서는 문제를 실제로 푸는 목적이 아니라 DSL용 구조 정리에 초점이 있으므로, 정답 판단은 별도 수행하지 않습니다.
- 원본에 이미 정답 그림과 결론 문장이 보이지만, 본 문서에서는 그것을 관찰 사실로만 옮깁니다.
