[문제 개요]
- 문제 번호는 `38.`로 보인다.
- 문장은 `다음 중 반지름이 3 cm인 원을 그릴 수 있도록 컴퍼스를 바르게 벌린 것을 선택하세요.`로 보인다.
- 선택형 문제이며, 컴퍼스와 눈금자 그림 3개 중에서 고르는 구성이다.
- 아래쪽에는 `(정답)` 표시가 붙은 작은 그림과 `(해설)` 문장이 함께 보인다.

[원본에 보이는 요소]
- 상단 왼쪽에 문제 번호 표기 `□ 38.`이 보인다.
- 본문 문장에 `3 cm`가 굵게/진하게 강조된 듯 보인다.
- 중앙에는 컴퍼스-눈금자 그림 선택지 3개가 가로로 배치되어 있다.
- 왼쪽 아래에는 `(정답)` 텍스트가 있고, 그 옆/아래에 작은 컴퍼스-눈금자 그림이 있다.
- 하단에는 `(해설)컴퍼스를 원의 반지름만큼 벌린 것을 찾습니다.`라는 문장이 보인다.
- 각 그림에는 파란색 눈금자, 회색 컴퍼스 본체, 검은 바늘 끝, 주황색 연필심/표시 도구가 보인다.
- 눈금자에는 `0, 1, 2, 3` 숫자가 보인다.

[수식/관계 구조]
- 이 문항은 등호로 이어지는 계산식이나 식 변형이 보이지 않는 그림 선택형 문제이다.
- 수치 관계는 `반지름 = 3 cm`라는 조건을 그림의 컴퍼스 벌림과 연결하는 구조로 해석된다.
- 보이는 관계는 “컴퍼스를 원의 반지름만큼 벌림”과 “눈금자 위의 간격”의 대응이다.
- 수식 변형 체인은 없고, 길이 비교/측정의 시각적 대응만 있다.

[등호별 변환 의미]
- 해당 없음.
- 등호 `=`를 포함한 변환식이 보이지 않는다.
- 계산 단계 설명이 필요한 문제 유형은 아니다.

[빈칸 정책]
- 눈에 보이는 빈칸은 없다.
- 정답 선택을 추론해서 채워 넣지 않는다.
- 정답 그림이 따로 보이더라도, 번호를 단정해 연결하지 않고 그대로 관찰 사실만 기록한다.
- 해상도 때문에 눈금 위치가 애매한 부분은 빈칸처럼 임의 보정하지 말고, 애매함을 유지한다.

[반복 패턴]
- 동일한 형태의 `컴퍼스 + 눈금자` 그림이 여러 번 반복된다.
- 각 선택지는 공통적으로 다음 요소를 가진다:
  - 아래에 파란색 직사각형 눈금자
  - 눈금자 위에 검은 눈금선과 숫자 `0, 1, 2, 3`
  - 위쪽에 회색 컴퍼스 몸체
  - 한쪽 끝의 검은 바늘
  - 다른 쪽 끝의 주황색 도구
- 반복되는 구조에서 다른 점은 컴퍼스 다리 사이의 벌어진 정도이다.
- 왼쪽 아래의 `(정답)` 그림도 같은 템플릿을 따르는 별도 반복 요소로 볼 수 있다.

[DSL 작성 힌트]
- 문제 텍스트와 선택지 그림들을 분리해서 배치하는 구성이 적절하다.
- 선택지는 3개의 독립된 그림 블록으로 처리하면 좋다.
- 정답 표시가 있는 작은 그림은 선택지와 별개의 보조 예시 블록으로 두는 것이 안전하다.
- 눈금자는 파란 바탕의 가로 직사각형으로 그리고, 검은 눈금과 숫자 `0~3`을 얹는 방식이 어울린다.
- 컴퍼스는 회색 본체, 바늘은 검은색, 연필심/표시 도구는 주황색으로 색을 구분하면 원본과 유사하다.
- 각 선택지에서 컴퍼스의 벌어짐 정도만 다르게 표현하고, 나머지 구성은 최대한 동일하게 유지하면 비교가 쉬워진다.
- `(정답)` 텍스트는 작은 그림의 왼쪽에 붙은 라벨처럼 배치되어 있다.
- `(해설)` 문장은 하단 텍스트 영역에 따로 둔다.
- 렌더링 시에는 그림들 사이 간격을 충분히 두고, 선택지 3개를 가로 정렬하는 것이 좋다.

[semantic/solvable 힌트]
- 문제의 의미는 “반지름 3 cm에 해당하는 컴퍼스 벌림”을 시각적으로 찾는 것이다.
- 눈금자에서 컴퍼스 두 끝이 얼마나 떨어져 있는지가 핵심 비교 요소다.
- 그림만으로는 선택지를 판정하지 말고, 원본에 있는 배치와 상대적 벌림 정도만 기록하는 것이 좋다.
- 해설 문장에 따르면 컴퍼스는 원의 반지름만큼 벌려야 한다는 의미가 드러난다.
- 다만 이 문서에서는 어느 선택지가 정답인지 추론하여 확정하지 않는다.
