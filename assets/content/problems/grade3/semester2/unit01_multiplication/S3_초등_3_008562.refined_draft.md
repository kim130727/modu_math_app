[문제 개요]
- 문제 번호는 `27.`로 보입니다.
- 지시문은 `계산 결과가 더 큰 것을 찾아 기호를 선택하세요.`입니다.
- 오른쪽 상단에는 보라색 테두리의 큰 직사각형 박스가 있고, 그 안에 두 계산식이 나란히 제시되어 있습니다.
- 왼쪽 아래에는 `(정답)`과 `(해설)`이 순서대로 배치되어 있으며, 해설 아래에 두 곱셈식의 계산 결과와 비교식이 제시됩니다.
- 전체 구조는 “두 계산 결과를 비교하여 더 큰 쪽의 기호를 고르는 선택형 문제”로 보입니다.

[원본에 보이는 요소]
- 상단 왼쪽에 문제 번호와 문장: `27. 계산 결과가 더 큰 것을 찾아 기호를 선택하세요.`
- 상단 오른쪽 박스 내부:
  - `① 4 × 93`
  - `ㄴ 7 × 54`
- 왼쪽 중간:
  - `(정답) ㄴ`
  - `(해설)`
- 해설 영역 왼쪽 계산식:
  - 위쪽 기호 `①`
  - 노란 배경 숫자 `1`
  - 세로 곱셈식 `4` / `× 93`
  - 아래 결과 `372`
- 해설 영역 오른쪽 계산식:
  - 위쪽 기호 `ㄴ`
  - 노란 배경 숫자 `2`
  - 세로 곱셈식 `7` / `× 54`
  - 아래 결과 `378`
- 해설 맨 아래:
  - `① < ㄴ`
- 화면 왼쪽 아래:
  - 회색 화살표 모양 아이콘

[수식/관계 구조]
- 보이는 수식은 두 개의 독립된 곱셈식입니다.
  - `4 × 93`
  - `7 × 54`
- 해설에서는 각 식이 세로 곱셈 형태로 재배치되어 있습니다.
  - 왼쪽: `4`가 위, `× 93`이 아래
  - 오른쪽: `7`이 위, `× 54`이 아래
- 각 세로 곱셈식 아래에 결과 숫자가 따로 적혀 있습니다.
  - 왼쪽 결과: `372`
  - 오른쪽 결과: `378`
- 마지막에는 두 기호를 비교하는 형태 `① < ㄴ`가 보입니다.
- 보이는 흐름은 `곱셈식 제시 → 계산 결과 표시 → 두 결과 비교`입니다.

[등호별 변환 의미]
- 화면에는 등호 `=`를 사용한 변형식이 연속으로 보이지 않습니다.
- 따라서 등호별 변환을 설명할 수 있는 체인은 확인되지 않습니다.
- 대신, 각 식은 세로 곱셈과 결과값이 분리되어 제시되는 구조입니다.
- 보이는 의미만 정리하면:
  - `4 × 93`의 결과가 `372`로 제시됨
  - `7 × 54`의 결과가 `378`로 제시됨
  - 이후 두 항목의 크기 비교로 이어짐
- 비교식 `① < ㄴ`은 결과 비교를 나타내는 최종 배치처럼 보이지만, 화면만으로 그 사용 맥락을 단정할 수는 없습니다.

[빈칸 정책]
- 이 원본에는 빈칸 입력란이 직접적으로 보이지 않습니다.
- 다만 DSL 작성 시에도, 이미지에 명시적으로 보이는 값만 기록하고 보이지 않는 값은 추론해서 채우지 않아야 합니다.
- 정답이 화면에 인쇄되어 보이는 경우에만 관찰 사실로 언급할 수 있습니다.
- 본 자료에서는 `(정답) ㄴ`이 보이므로, 이것은 “보이는 사실”로만 기록합니다.
- 빈칸이 존재하는 레이아웃으로 변환할 때도, 추정값을 넣지 말고 빈칸은 빈칸 상태로 유지해야 합니다.

[반복 패턴]
- 두 계산식이 거의 같은 형식으로 반복됩니다.
  - 위쪽에 기호 표시
  - 노란색 배경의 번호 라벨 `1`, `2`
  - 세로 곱셈식
  - 아래쪽 결과 숫자
- 왼쪽과 오른쪽의 배치 방식이 대응 구조를 이룹니다.
- 상단 박스 안의 두 항목도 좌우 대칭적으로 배치되어 있습니다.
- 해설의 두 식은 각각 상단 박스의 두 선택지와 대응되는 반복 패턴으로 보입니다.
- 왼쪽 식과 오른쪽 식 모두 같은 간격, 같은 정렬, 같은 시선 흐름을 갖도록 구성된 화면입니다.

[DSL 작성 힌트]
- 상단의 문제 문장과 오른쪽 상단 박스는 서로 분리된 텍스트/도형 블록으로 다루는 것이 좋습니다.
- 보라색 테두리 박스는 얇은 선의 직사각형으로 표현하고, 내부 두 항목은 좌우 균등 간격으로 배치하세요.
- 상단 박스 안의 각 항목은 `기호 + 곱셈식`을 한 줄에 놓는 방식이 적합합니다.
- 해설 영역은 좌우 두 칼럼으로 나누어, 각 칼럼에 세로 곱셈식과 결과 숫자를 위아래로 정렬하세요.
- 노란 배경 숫자 `1`, `2`는 계산식 위쪽의 작은 강조 라벨로 따로 배치하세요.
- 결과 숫자 `372`, `378`은 각 계산식의 하단 중앙에 배치하면 원본과 유사합니다.
- 왼쪽 아래의 회색 화살표는 별도 아이콘 요소로 두고, 비교식 `① < ㄴ`과 가까운 위치에 놓으세요.
- `①`, `ㄴ` 같은 기호는 글자 크기와 위치가 결과와 혼동되지 않도록 분리 배치하는 것이 좋습니다.
- 세로 곱셈식에서는 숫자와 `×`가 줄바꿈되어 보이므로, 분수 슬롯처럼 위/아래 정렬을 명확히 해주세요.

[semantic/solvable 힌트]
- 화면에 보이는 계산식은 두 개이며, 각각의 결과 숫자도 화면에 직접 제시되어 있습니다.
- 보이는 사실만으로는 `372`와 `378`의 계산 과정 전체를 다시 전개할 필요는 없습니다.
- 문제는 “더 큰 계산 결과를 가진 기호를 고르기” 유형으로 이해됩니다.
- 다만 여기서는 정답을 새로 추론하지 않고, 화면에 인쇄된 `(정답) ㄴ`만 관찰 사실로 기록합니다.
- 따라서 semantic 레벨에서는 “두 곱셈식의 결과 비교 후 기호 선택” 정도까지만 기술하면 충분합니다.
