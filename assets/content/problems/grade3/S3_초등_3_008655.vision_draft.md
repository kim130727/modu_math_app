[전체적인 구성]
- 화면 상단에 문제 번호와 지시문이 한 줄/두 줄로 배치되어 있습니다.
- 가운데 오른쪽 영역에 원 모양의 도형이 크게 그려져 있고, 원의 중심점에서 여러 점으로 선분이 뻗어 있습니다.
- 원 아래에는 선택 문장이 한 줄로 제시되어 있으며, 그 아래에 `(정답)`과 `(해설)`이 텍스트로 이어집니다.
- 전체적으로 “문장 선택형 + 도형 설명” 구조입니다.
- 이 문제는 답을 직접 쓰는 칸보다는, 괄호 안의 선택지에서 알맞은 말을 고르는 형식으로 보입니다.

[보이는 텍스트]
- `□ 30.`
- `선분 ㅇㄱ, 선분 ㅇㄴ, 선분 ㅇㄷ은 모두 원의 반지름입니다. 각 선의 길이를 재어 알맞은 말을 선택하세요.`
- `한 원에서 원의 반지름은 모두 ( 같습니다, 다릅니다 ).`
- `(정답) 같습니다`
- `(해설) (선분 ㅇㄱ) = (선분 ㅇㄴ) = (선분 ㅇㄷ) = 2 cm`
- `따라서 세 선분의 길이는 모두 같으므로 한 원에서 원의 반지름은 모두 같습니다.`
- 도형 안의 표기:
  - 중심점 근처에 `ㅇ`로 보이는 글자/기호가 있음
  - 원 둘레 쪽 점들에 `ㄱ`, `ㄴ`, `ㄷ`로 보이는 글자가 붙어 있음

[반복되는 시각 패턴]
- 하나의 중심점에서 여러 선분이 방사형으로 뻗는 형태가 반복적으로 나타납니다.
- 원의 중심에서 원 위의 점으로 이어지는 선분이 3개 보이며, 모두 같은 종류의 요소(반지름)라는 점을 시각적으로 비교하게 합니다.
- 본문 문장 안에 두 개의 선택어 `( 같습니다, 다릅니다 )`가 같은 형식의 괄호 안에 나란히 제시되어 있습니다.
- 해설 부분에서는 `=` 기호가 연속해서 사용되어 세 선분의 길이가 같다는 점을 나열형으로 표현합니다.

[수식 단계] (해당될 때)
- 해설에 등호가 연속해서 등장합니다.
- 보이는 형태: `(선분 ㅇㄱ) = (선분 ㅇㄴ) = (선분 ㅇㄷ) = 2 cm`
- 각 등호는 앞의 선분 길이와 다음 선분 길이를 같은 값으로 연결하는 나열식 표현으로 보입니다.
- 마지막에 `= 2 cm`가 붙어 있어, 세 선분의 길이가 같은 하나의 수치로 제시되어 있습니다.
- 다만 이 초안에서는 그 수치의 타당성이나 의미를 계산해서 설명하지 않고, 화면에 적힌 형태만 기록합니다.

[등호별 변환 의미] (해당될 때)
- 첫 번째 `=`: `(선분 ㅇㄱ)`과 `(선분 ㅇㄴ)`의 길이가 같다는 표시로 보입니다.
- 두 번째 `=`: `(선분 ㅇㄴ)`과 `(선분 ㅇㄷ)`의 길이가 같다는 표시로 보입니다.
- 세 번째 `=`: 세 선분의 길이가 하나의 값 `2 cm`와 같다고 적은 부분입니다.
- 즉, 화면에서는 여러 반지름의 길이를 순서대로 같다고 연결해 보여 주는 형식입니다.

[세부 내용]
- 도형은 원 1개로 보이며, 중심점은 원의 내부 중앙보다 약간 오른쪽/아래쪽에 위치한 작은 점으로 표시되어 있습니다.
- 중심점에서 원 둘레의 세 지점으로 선분이 이어져 있습니다.
  - 왼쪽 아래 방향으로 뻗는 선분 1개
  - 오른쪽 아래 방향으로 뻗는 선분 1개
  - 오른쪽 방향으로 약간 위쪽으로 뻗는 선분 1개
- 선분 끝점들은 원 둘레 위에 놓여 있습니다.
- 선분의 끝에 각각 `ㄱ`, `ㄴ`, `ㄷ`로 보이는 표기가 붙어 있어 점 이름을 나타내는 것으로 보입니다.
- 중심점 근처의 `ㅇ` 표시는 중심을 뜻하는 기호처럼 사용된 것으로 보이지만, 글자 형태가 다소 흐릿합니다.
- 원은 얇은 검은 선으로 그려져 있고, 선분도 같은 굵기의 선으로 그려져 있습니다.
- 선택 문장은 원 아래 중앙에 배치되어 있습니다.
- `(정답)`과 `(해설)`은 왼쪽 아래 쪽에 작은 글씨로 따로 구분되어 있습니다.

[Python DSL 작성 힌트]
- 도형 생성 시:
  - 원 1개
  - 중심점 1개
  - 중심에서 원둘레의 3점으로 이어지는 선분 3개
- 텍스트 배치 시:
  - 상단 문제 문장 1개
  - 원 아래 선택 문장 1개
  - 하단에 정답/해설 텍스트 2개 블록
- 선택지 표시는 괄호 안에 두 단어가 쉼표로 구분된 형태로 표현하면 됩니다.
- 해설의 등호 나열식은 수식 객체 또는 일반 텍스트 라인으로 `A = B = C = D` 형식으로 배치하면 됩니다.
- 도형의 점 이름은 위치에 맞게 원 둘레 방향으로 붙여야 합니다.
- 원과 선분의 상대적 위치가 중요하므로, 좌우/상하 배치를 맞춰서 그리는 것이 좋습니다.

[주의할 점]
- 문제를 푸는 과정이나 정답 추론은 넣지 마세요.
- `( 같습니다, 다릅니다 )`에서 어떤 말을 선택해야 하는지 추가로 판단해 쓰지 마세요.
- `ㅇㄱ`, `ㅇㄴ`, `ㅇㄷ` 표기는 실제 글자처럼 보이지만, OCR 오독 가능성이 있으므로 그대로 단정하기보다 이미지에 보이는 대로 옮기는 것이 안전합니다.
- 중심점의 기호 `ㅇ`도 정확한 문자 판독이 다소 애매할 수 있습니다.
- 하단 해설의 `2 cm`는 화면에 보이는 텍스트를 옮긴 것이지, 계산 결과를 설명하는 것이 아닙니다.

[불확실한 부분]
- 중심점 표기가 `ㅇ`인지 다른 기호인지 약간 흐릿합니다.
- 원 둘레의 점 이름이 `ㄱ, ㄴ, ㄷ`로 보이지만, OCR 관점에서는 다른 자모로 읽힐 가능성이 있습니다.
- 상단 문제 번호 앞의 작은 네모(`□`)는 체크박스처럼 보이지만, 인쇄물의 장식 요소일 수도 있습니다.
- 해설 문장 속 `(선분 ㅇㄱ)` 등의 표기가 정확히 괄호 포함인지, 또는 OCR 과정에서 괄호가 일부 추가/생략되었는지 약간 불확실합니다.
