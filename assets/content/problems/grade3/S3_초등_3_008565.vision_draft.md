# [전체적인 구성]
- 상단에 문제 번호와 지시문이 한 줄로 배치되어 있음.
- 중앙~우측 상단에는 구름 모양 테두리 안에 곱셈식들이 각각 따로 제시되어 있음.
- 왼쪽 아래에는 해설 형태로 각 곱셈식의 계산 결과와 `2000`을 비교한 문장이 세로로 나열되어 있음.
- 가장 아래 왼쪽에는 작은 구름 모양이 하나 더 보이며, 그 안에 곱셈식이 다시 한 번 들어 있음.
- 전체적으로 “여러 식 중 계산 결과가 2000보다 큰 것 찾기” 유형의 시각 자료로 보임.

# [보이는 텍스트]
- `30.`  
- `계산 결과가 2000보다 큰 곱셈식을 선택해 보세요.`
- 구름 안 수식들:
  - `19 × 90`
  - `20 × 70`
  - `40 × 50`
  - `36 × 60`
- 해설 텍스트:
  - `(해설)19 × 90 = 1710 < 2000`
  - `20 × 70 = 1400 < 2000`
  - `40 × 50 = 2000 = 2000`
  - `36 × 60 = 2160 > 2000`
- 아래쪽 작은 구름:
  - `36 × 60`

# [반복되는 시각 패턴]
- 같은 형태의 구름(말풍선 비슷한 윤곽) 안에 곱셈식이 1개씩 들어가는 패턴이 반복됨.
- 해설 영역에서는 각 줄이 `곱셈식 = 결과 비교기호 2000` 구조로 반복됨.
- 비교기호는 `<`, `=`, `>`가 각각 사용되어, 결과와 2000의 관계를 시각적으로 보여줌.
- 곱셈식 표기에서 `×` 기호가 반복 사용됨.

# [수식 단계]
- 화면에 보이는 해설은 각각 한 줄씩 독립된 비교식 형태임.
- 별도의 연쇄적인 등호 변형 과정은 보이지 않음.
- 각 줄은 `곱셈식 = 계산 결과` 이후 `비교기호 2000`으로 연결된 형태로 읽힘.
- 예:
  - `19 × 90 = 1710 < 2000`
  - `20 × 70 = 1400 < 2000`
  - `40 × 50 = 2000 = 2000`
  - `36 × 60 = 2160 > 2000`

# [등호별 변환 의미]
- `=`는 곱셈식의 계산 결과를 적어 놓은 것으로 보임.
- 뒤에 이어지는 `<`, `=`, `>`는 그 결과와 `2000`의 크기를 비교한 표시로 보임.
- 즉, 각 줄은 “곱셈 결과를 구한 뒤 2000과 비교”하는 의미 구조로 보임.
- 다만 실제 문제에서 어떤 식을 선택하라는 것인지, 정답 표시가 따로 있는지는 화면만으로는 명시되지 않음.

# [세부 내용]
- 문제 번호 `30.`은 왼쪽 상단에 작게 표시되어 있음.
- 지시문은 문제 번호 오른쪽에 길게 적혀 있으며, 끝에 마침표가 있음.
- 상단의 구름 모양 4개는 화면의 위쪽과 오른쪽 위쪽에 흩어져 배치되어 있음.
  - 왼쪽 위 쪽에 `19 × 90`
  - 오른쪽 위 쪽에 `20 × 70`
  - 가운데 오른쪽 아래 쪽에 `40 × 50`
  - 더 오른쪽 아래 쪽에 `36 × 60`
- 해설은 왼쪽 중하단에 세로 방향으로 정렬되어 있음.
- 해설 각 줄은 왼쪽에 곱셈식, 가운데에 비교기호, 오른쪽에 `2000`이 오는 구조임.
- 아래쪽 작은 구름 `36 × 60`은 왼쪽 맨 아래에 일부만 보이는 듯한 배치로, 상단 구름과 같은 서식을 가진 반복 요소로 보임.
- 일부 수식은 작은 구름 안에 있고, 해설에서는 같은 수식이 텍스트로 다시 등장함.

# [Python DSL 작성 힌트]
- 상단 제목 영역, 구름형 수식 박스 여러 개, 왼쪽 하단 해설 목록을 분리해서 배치하면 좋음.
- 각 구름형 요소는 동일한 스타일의 도형으로 정의하고, 내부 텍스트만 다르게 넣는 방식이 적합해 보임.
- 해설 영역은 세로 목록 형태로, 각 줄마다
  - 왼쪽: 곱셈식
  - 가운데: 비교기호
  - 오른쪽: `2000`
  의 3단 구조로 배치하면 원본과 유사함.
- 문제 지시문은 상단 한 줄 텍스트로 처리.
- 아래쪽 작은 구름은 화면 하단 일부에 걸쳐 보이도록 클리핑 또는 부분 노출 형태로 처리하면 유사함.
- 정답 표시용 체크박스나 선택 표시가 보이지 않으므로, 현재 화면만 기준으로는 추가 선택 UI를 넣지 않는 편이 안전함.

# [주의할 점]
- 계산 결과를 직접 추론해 정답을 채우지 말 것.
- `2000보다 큰 곱셈식`이라는 문장을 보고 선택 결과를 단정적으로 적지 말 것.
- 아래쪽 작은 구름의 내용이 상단과 같아 보여도, 시각적으로는 일부만 보이는 요소이므로 별도 객체로 다루는 것이 좋음.
- 해설의 비교기호는 등호 변형 과정이 아니라 비교 표현이므로, 일반적인 연립 변형식처럼 해석하지 말 것.
- 구름 도형의 정확한 윤곽선은 둥근 물결형이며, 일반 직사각형 박스와 다름.

# [불확실한 부분]
- 하단 왼쪽의 작은 구름이 원래 의도한 선택 표시인지, 장식용 반복 요소인지 화면만으로는 확정하기 어려움.
- 해설 영역의 `(해설)` 표시는 실제 인쇄 요소인지, 편집상 덧붙은 주석인지 단정하기 어려움.
- 상단 구름들의 정확한 좌표와 크기는 이미지 해상도상 대략적 추정만 가능함.
- 비교기호 주변의 간격과 글자 크기는 원본 구현 시 약간 다를 수 있음.
