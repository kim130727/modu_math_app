[문제 개요]
- 문제 번호: `24.`
- 지시문: `다음 계산에서 3 × 8 = 24의 4를 써야 할 곳을 찾아 기호를 선택하세요.`
- 화면 구성은 크게 4부분으로 보임:
  1. 상단의 문제 문장
  2. 중앙 상단의 초록색 테두리 작은 박스
  3. 왼쪽 중간의 `(정답)` 표기
  4. 하단 왼쪽의 `(해설)` 영역
- 보이는 사실과 추론을 분리하면:
  - 보이는 사실: 숫자와 기호, 세로셈 형태, 화살표, 원형 기호가 배치되어 있음.
  - 추론되는 수학적 의미: 자리값에 맞는 숫자 위치를 고르는 선택형 문제로 보임.
- 정답이 화면에 인쇄되어 보이지만, 여기서는 사실로만 기록하고 그 의미는 확장하지 않음.

[원본에 보이는 요소]
- 상단 왼쪽:
  - `24.`
  - `다음 계산에서 3 × 8 = 24의 4를 써야 할 곳을 찾아 기호를 선택하세요.`
- 중앙 상단의 초록색 테두리 박스:
  - 연두색/초록색 가는 테두리의 직사각형
  - 내부에 세로셈 형태의 배열
  - 위 줄: `3 0`
  - 아래 줄: `× 8 0`
  - 가로선 아래: 원 안의 기호 `ㄱ`, `ㄴ`, `ㄷ`, `ㄹ`이 좌→우로 배치
- 왼쪽 중간:
  - `(정답) ㄴ`
- 하단 왼쪽의 해설 영역:
  - `(해설)`
  - 같은 세로셈이 더 크게 다시 제시됨
  - `3 0`
  - `× 8 0`
  - 가로선
  - `2 4 0 0`
  - 결과 숫자 각 자리 아래에 위쪽 화살표가 4개
  - 맨 아래에 원 안의 `ㄱ`, `ㄴ`, `ㄷ`, `ㄹ`이 가로로 배치
- 원형 기호:
  - 모두 동그란 테두리 안에 한 글자씩 들어 있음
  - 보기 역할로 보이며, 작은 박스와 해설 영역 모두에 반복됨

[수식/관계 구조]
- 문제 문장 안에 `3 × 8 = 24`가 텍스트로 직접 등장함.
- 해설 영역의 세로셈은 다음처럼 보임:
  - 윗줄: `3 0`
  - 아랫줄: `× 8 0`
  - 결과: `2 4 0 0`
- 구조상 자리값을 맞춘 곱셈 세로셈 도식으로 보임.
- 다만 화면에는 중간 과정, 부분곱, 자리올림 표시가 따로 보이지 않음.
- 결과 아래의 화살표 4개는 각 자리와 1:1 대응되는 안내 표시처럼 배치되어 있음.

[등호별 변환 의미]
- 화면에 `=` 기호가 연쇄적으로 여러 번 나타나는 수식 전개는 보이지 않음.
- 문제 문장 속 `3 × 8 = 24`는 하나의 진술로 보이며, 별도의 계산식 변형 단계는 없음.
- 따라서 등호별 단계 설명은 적용 대상이 거의 없음.
- 해설 영역도 결과를 보여주는 세로셈 도식일 뿐, `=`, `→` 등의 변형 연쇄는 보이지 않음.

[빈칸 정책]
- 화면에 보이는 빈칸 또는 자리 선택 요소는 추론값으로 채우지 않음.
- 보기에 해당하는 `ㄱ, ㄴ, ㄷ, ㄹ` 위치는 그대로 보이는 대로 유지해야 함.
- `(정답) ㄴ`은 이미지에 인쇄되어 보이는 사실이므로 기록 가능하지만, 그 의미를 더 확장하여 서술하지 않음.
- 빈칸이 있다면 DSL에서는 빈칸으로 남겨 두는 것이 원칙이며, 임의 보완은 금지.
- 이 문제는 보기 선택형이므로, 보이는 기호 배열과 위치를 그대로 보존하는 것이 중요함.

[반복 패턴]
- 작은 박스와 해설 영역 모두 동일한 세로셈 레이아웃을 반복 사용함.
- 숫자 `3`, `0`, `8`, `0`은 자릿수 정렬을 위해 공백을 둔 채 배치됨.
- 원형 보기 `ㄱ, ㄴ, ㄷ, ㄹ`이 두 군데 모두 반복됨.
- 해설 영역에서는 결과 `2400` 아래에 화살표가 반복적으로 4개 배치되어, 각 자리와 대응되는 패턴을 형성함.
- 반복 요소는 다음처럼 묶어 인식하면 좋음:
  - 문제 문장
  - 작은 세로셈 박스
  - 정답 표시
  - 큰 해설 세로셈

[DSL 작성 힌트]
- 상단 문장, 초록색 박스, 정답 표기, 해설 영역을 서로 분리된 블록으로 배치하는 구성이 적합함.
- 세로셈은 텍스트 줄만으로 처리하기보다, 자릿수 정렬이 드러나도록 행 단위 또는 좌표 기반 배치를 사용하는 편이 좋음.
- 원형 기호 `ㄱ, ㄴ, ㄷ, ㄹ`은 동일한 크기와 간격으로 가로 배열해야 함.
- 초록색 박스는 테두리만 있는 작은 사각형으로 표현하고, 내부 요소는 가운데 정렬에 가깝게 배치하는 것이 자연스러움.
- 해설 영역의 `2400` 아래 화살표는 각 자리 숫자와 1:1로 맞춰 배치해야 함.
- `(정답)`은 텍스트와 기호가 인접해 있으므로, 라벨과 선택 기호를 붙여서 배치하는 것이 좋음.
- 작은 박스의 가로선과 해설 영역의 가로선 길이는 서로 달라 보일 수 있으므로 별도로 처리하는 것이 안전함.

[semantic/solvable 힌트]
- 문제는 “4를 써야 할 곳”을 고르는 형태의 선택 문제로 보임.
- 해설 영역에 `2400`이 제시되어 있어, 자리 대응 관계를 보여 주려는 의도로 읽힐 수 있음.
- 다만 여기서는 풀이를 확정하거나 어떤 자리에 4가 들어가는지 직접 결론내리지 않음.
- 보이는 정보만으로는 “어느 자리”를 설명하는 도식적 안내가 핵심이며, 정답 의미는 추가 추론하지 않는 것이 원칙임.
