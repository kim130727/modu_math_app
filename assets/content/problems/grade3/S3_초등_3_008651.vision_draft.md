[전체적인 구성]
- 화면 상단에 문제 번호와 지시문이 왼쪽부터 가로로 배치되어 있다.
- 본문에는 컴퍼스 그림이 여러 개 제시되어 있으며, 위쪽에 3개가 가로로 나란히 놓여 있고, 왼쪽 아래에 1개가 따로 배치되어 있다.
- 왼쪽 아래 그림 옆에는 `(정답)` 표시가 있어 정답 위치를 가리키는 형식이다.
- 화면 하단에는 `(해설)`로 시작하는 문장형 설명이 한 줄 이상 이어진다.
- 전체적으로 “보기 그림 3개 + 정답 그림 1개 + 해설 문장” 구조이다.

[보이는 텍스트]
- `□ 24. 컴퍼스를 3 cm가 되도록 벌린 것을 찾아 선택해 보세요.`
- `(정답)`
- `(해설) 컴퍼스의 침을 자리 눈금 0에 맞추고 연필심을 자리 눈금 3에 오도록 컴퍼스를 벌린 것을 찾습니다.`
- 숫자 표기:
  - 각 자 그림에 `0, 1, 2, 3` 눈금 숫자가 보인다.
- 단위 표기:
  - 문제 문장에 `3 cm`가 보인다.

[반복되는 시각 패턴]
- 같은 형태의 컴퍼스 그림이 여러 번 반복된다.
- 각 그림은 거의 동일한 구조를 가지며,
  - 위쪽에 컴퍼스 본체
  - 아래쪽에 파란색 자
  - 자 위에 눈금과 숫자 `0~3`
  - 컴퍼스의 한 다리는 자의 `0` 근처, 다른 다리는 `3` 근처에 위치한 모습
- 위쪽 3개의 그림은 서로 비슷한 선택지처럼 보이고, 아래 왼쪽의 그림은 `(정답)` 표기와 함께 별도로 강조되어 있다.
- 컴퍼스의 한쪽 끝은 바늘처럼 뾰족하고, 다른 쪽 끝은 연필심처럼 색이 있는 막대 형태로 표현되어 있다.

[수식 단계]
- 해당 없음. 수식 변형식이나 등호가 연속되는 표현은 보이지 않는다.

[등호별 변환 의미]
- 해당 없음. 등호가 포함된 단계적 변환은 보이지 않는다.

[세부 내용]
- 각 그림에는 파란색 자가 가로로 놓여 있고, 자의 아래쪽에 짙은 파란 테두리와 눈금선이 있다.
- 자의 왼쪽 끝에는 `0`, 오른쪽 방향으로 `1, 2, 3` 숫자가 표시되어 있다.
- 컴퍼스는 회색/파란색 계열의 금속 다리와 위쪽 손잡이, 한쪽 다리의 끝에 연필심처럼 보이는 노란색 막대와 갈색 끝 부분이 결합된 형태로 그려져 있다.
- 컴퍼스의 바늘 끝은 자의 `0` 부근에 닿아 있고, 연필심 쪽 끝은 자의 `3` 부근에 닿아 있는 장면이 반복적으로 보인다.
- 위쪽 3개의 그림은 위치나 벌어진 정도가 거의 같아 보이나, 인쇄 상태상 미세한 차이는 있을 수 있다.
- `(정답)` 표시는 왼쪽 아래 그림 왼편에 붙어 있어 다른 그림보다 답임을 직접 알려준다.
- 해설 문장은 문제 풀이를 설명하는 참고 문장으로, 눈금을 `0`과 `3`에 맞추는 동작을 서술한다.

[Python DSL 작성 힌트]
- 문제 텍스트 영역과 그림 영역을 분리해서 배치하는 구조가 적절하다.
- 상단 문제 문장은 한 줄 텍스트로 넣고, 숫자 `24`와 `3 cm`가 눈에 띄도록 표현한다.
- 컴퍼스 그림은 선택지 3개를 상단에 가로 정렬하고, 정답 그림 1개를 좌하단에 따로 배치하는 식으로 구성하면 된다.
- 각 그림 아래에는 자 도형이 있고, `0~3` 숫자와 눈금선이 반복되므로 이를 재사용 가능한 패턴으로 만들 수 있다.
- `(정답)` 텍스트는 좌하단 그림 왼쪽에 별도 라벨처럼 넣는 것이 좋다.
- 해설 문장은 하단에 작은 글씨 또는 일반 본문 크기로 배치한다.
- 자의 눈금은 세밀한 작은 선들이 반복되는 형태이므로, 단순 숫자만 넣지 말고 눈금 막대의 반복성도 반영하면 좋다.

[주의할 점]
- 답을 추론해서 특정 선택지 번호를 단정해 적지 말아야 한다.
- 그림의 정확한 벌어진 각도나 길이를 수치로 계산해 서술하지 말아야 한다.
- `3 cm`는 문제 조건으로 보이는 텍스트일 뿐, 계산 결과가 아니다.
- 컴퍼스의 실제 비율은 이미지상 약간 왜곡되어 보일 수 있으므로 너무 엄밀한 기하로 해석하지 않는 것이 좋다.
- 위쪽 3개 그림이 모두 거의 같아 보여도, 단정적으로 “같다”고만 쓰기보다 “비슷해 보인다”라고 표현하는 것이 안전하다.

[불확실한 부분]
- 위쪽 3개의 그림이 각각 서로 완전히 동일한지, 아니면 미세하게 다른지 이미지만으로는 확정하기 어렵다.
- 컴퍼스 다리의 정확한 접점 위치가 자의 어떤 세부 눈금에 정확히 맞는지는 이미지 해상도상 다소 अस्प명확하다.
- `(정답)` 표시가 실제 정답 번호를 뜻하는지, 아니면 예시 이미지의 강조인지 문맥상 정답 안내로 보이지만, DSL에서는 라벨로만 처리하는 것이 안전하다.
