# [문제 개요]
- 문제 번호는 `79.`로 보인다.
- 지시문은 `원의 반지름과 지름을 나타내는 선분을 각각 찾아 기호를 선택해 보세요.`로 읽힌다.
- 중앙에는 원 그림이 있고, 원 안의 선분들과 기호 `ㄱ, ㄴ, ㄷ`를 보고
  - `원의 반지름`
  - `원의 지름`
  에 해당하는 기호를 고르는 유형이다.
- 하단에는 선택형 문장과 함께 `(정답)`, `(해설)` 표기가 보인다.
- **보이는 사실**: 정답과 해설 텍스트가 화면에 인쇄되어 있다.
- **추론되는 수학적 의미**: 그림 속 선분들을 반지름/지름으로 분류하는 문항이다.

# [원본에 보이는 요소]
- 상단 왼쪽에 `79.`가 있다.
- 상단 지시문:
  - `원의 반지름과 지름을 나타내는 선분을 각각 찾아 기호를 선택해 보세요.`
- 중앙 도형:
  - 원 1개
  - 원 내부의 작은 점 1개
  - 원 내부의 직선 선분 여러 개
  - 파란색 점선 곡선(호처럼 보이는 보조 표시) 여러 개
  - 기호 `ㄱ`, `ㄴ`, `ㄷ`가 원 내부의 서로 다른 위치에 표시됨
- 원 아래 문장:
  - `원의 반지름 ( ㄱ, ㄴ, ㄷ ), 원의 지름 ( ㄱ, ㄴ, ㄷ )`
- 좌하단 표기:
  - `(정답) ㄴ, ㄱ`
  - `(해설) 원의 반지름 ㄴ, 원의 지름 ㄱ`
- **보이는 사실**: 선택지는 `ㄱ, ㄴ, ㄷ` 세 개이며, 문장 안의 괄호와 쉼표 형식이 유지되어 있다.
- **보이는 사실**: 해설 줄에는 `원의 반지름 ㄴ, 원의 지름 ㄱ`가 인쇄되어 있다.

# [수식/관계 구조]
- 등호나 식 변형은 보이지 않는다.
- 계산 구조도 없다.
- 구조는 다음과 같이 읽힌다.
  - `원의 반지름` → 선택지 `ㄱ, ㄴ, ㄷ` 중 하나 고르기
  - `원의 지름` → 선택지 `ㄱ, ㄴ, ㄷ` 중 하나 고르기
- 원 내부의 기호와 선분을 대응시키는 분류형 구조이다.
- **추론되는 수학적 의미**: 중심에서 원 위까지의 선분은 반지름, 원을 가로지르는 선분은 지름일 가능성이 있다.
- 다만, 이 대응은 그림 해석에 해당하므로 정답을 새로 계산해 단정하지 않는다.

# [등호별 변환 의미]
- 해당 없음.
- 등호 연쇄나 식 변환 과정이 없다.

# [빈칸 정책]
- 이 문항에는 계산 빈칸이 아니라 선택지형 괄호가 있다.
- 원문에 보이는 `(` `ㄱ, ㄴ, ㄷ` `)` 형식은 그대로 유지한다.
- **추론값으로 빈칸을 채우지 않는다.**
- 이미지와 초안이 다를 경우, **이미지에 보이는 기호와 문장 형식**을 우선한다.
- 정답이 화면에 인쇄되어 보이더라도, DSL 작성용 초안에서는 이를 빈칸 대체값으로 쓰지 않는다. 관찰 사실로만 기록한다.

# [반복 패턴]
- `원의 반지름`과 `원의 지름`이라는 두 항목이 같은 형식으로 반복된다.
- 각 항목 뒤에 동일한 선택지 집합 `ㄱ, ㄴ, ㄷ`가 붙는다.
- 원 내부에는 여러 선분이 겹쳐 있으며, 기호도 도형 근처에 각각 배치되어 있다.
- 파란 점선 호가 원의 바깥쪽 또는 가장자리 근처를 따라 일부 구간에 나타난다.
- 반복 패턴은 “문항 문장 + 원 그림 + 선택지 2개 + 정답/해설” 구조로 정리할 수 있다.

# [DSL 작성 힌트]
- 레이아웃은 다음 순서로 나누면 좋다.
  1. 상단: 문제 번호 `79.`와 지시문
  2. 중앙: 원 도형 1개
  3. 원 내부: 선분들, 중심점처럼 보이는 점, 파란 점선 호, 기호 라벨 `ㄱ/ㄴ/ㄷ`
  4. 하단: `원의 반지름 ( ㄱ, ㄴ, ㄷ ), 원의 지름 ( ㄱ, ㄴ, ㄷ )`
  5. 좌하단: `(정답)`, `(해설)` 텍스트
- 원 도형은 중심점 주변의 선분들이 서로 겹치므로, 외곽 원 → 내부 선분 → 점선 보조선 → 기호 라벨 순으로 배치하면 읽기 쉽다.
- 선택지 문장은 쉼표와 괄호가 중요하므로 그대로 재현한다.
- `ㄱ, ㄴ, ㄷ`는 일반 문자보다 작은 원형 마커 안의 글자처럼 보이므로, 라벨 스타일을 따로 두는 것이 좋다.
- 원 안의 세 선분은 서로 다른 방향과 길이로 보이므로, 각 선분을 개별 요소로 분리해 배치하는 것이 적절하다.
- 중심처럼 보이는 작은 점은 원 내부의 기준점 요소로 두되, 중심이라고 단정하는 주석은 약하게 처리한다.

# [semantic/solvable 힌트]
- **보이는 사실**: 해설 줄에 `원의 반지름 ㄴ, 원의 지름 ㄱ`라고 적혀 있다.
- **보이는 사실**: 정답 줄에 `(정답) ㄴ, ㄱ`이 보인다.
- **추론되는 수학적 의미**: 문항은 원의 반지름과 지름을 구별하는 기초 도형 문제이다.
- 그림을 보면 한 선분은 중심에서 원둘레까지 닿는 형태로 보이고, 다른 선분은 원을 가로지르는 형태로 보인다.
- 그러나 이 문항의 풀이를 새로 전개할 필요는 없으며, 원본에 인쇄된 해설은 관찰 사실로만 기록한다.

# [주의 및 TODO]
- 원 내부 선분의 정확한 시작점/끝점은 해상도에 따라 약간 흐릿할 수 있다.
- 가운데의 점이 정확히 중심인지, 단순 표시인지 텍스트로 명시되어 있지 않다.
- 파란 점선 호는 보조 표시처럼 보이지만, 장식인지 의미 표시인지 단정하지 말고 TODO로 남겨도 된다.
- `ㄱ`, `ㄴ`, `ㄷ`의 상대 위치는 그림과 약간의 오차가 있을 수 있으므로, 배치 시 선분과의 겹침을 엄격하게 고정하지 말고 유연하게 처리한다.
- 등식 변형이나 분수 변환은 없으므로 관련 DSL 구조는 불필요하다.
- 정답과 해설이 화면에 보이지만, DSL 초안에서는 이를 풀이 결과로 확장하지 말고 텍스트 요소로만 취급한다.
