[문제 개요]
- 문제 번호는 `99.`로 보인다.
- 지시문은 `묶이 더 작은 것을 선택하세요.`로 인쇄되어 있다.
- 오른쪽 위에 분홍색 테두리의 큰 직사각형 박스가 있고, 그 안에 두 개의 나눗셈 식이 나란히 제시된다.
- 박스 아래에는 `(정답)` 표시와 함께 한 식이 한 줄로 제시된다.
- 그 아래에는 `(해설)` 표시와 함께 두 식의 계산 결과와 비교 문장이 이어진다.
- 전체 구성은 `비교 대상 2개 → 정답 표시 → 해설`의 순서로 읽힌다.

[원본에 보이는 요소]
- 왼쪽 위에 작은 네모 모양 기호가 있고, 그 옆에 `99.`가 있다.
- 지시문 텍스트: `묶이 더 작은 것을 선택하세요.`
- 분홍색 테두리 박스 안:
  - `96 ÷ 3`
  - `84 ÷ 2`
- 박스 아래:
  - `(정답)96 ÷ 3`
- 해설 부분:
  - `(해설)96 ÷ 3 = 32, 84 ÷ 2 = 42`
  - `따라서 32<42이므로 묶이 더 작은 것은 96 ÷ 3입니다.`
- 보이는 그대로의 표기이므로, `묶이`라는 문자열은 수정하지 않는다.
- `(정답)`과 `(해설)`은 본문과 같은 줄에서 시작하는 라벨처럼 보인다.
- 전체 배경은 흰색이고, 글자는 검정색 중심이다.
- 비교 박스만 연한 분홍색 선으로 강조되어 있다.

[수식/관계 구조]
- 비교 대상은 두 개이다:
  - 첫째 식: `96 ÷ 3`
  - 둘째 식: `84 ÷ 2`
- 해설에서는 각 식의 결과가 다음과 같이 함께 제시된다:
  - `96 ÷ 3 = 32`
  - `84 ÷ 2 = 42`
- 이어서 두 결과를 비교하는 식이 나온다:
  - `32<42`
- 마지막 문장은 비교 결과를 근거로 선택 대상을 말한다.
- 이 화면에서 직접 보이는 수식 구조는 `나눗셈 식 → 결과 → 두 값 비교 → 결론 문장`이다.

[등호별 변환 의미]
- `96 ÷ 3 = 32`
  - 왼쪽의 나눗셈 식에 대해 오른쪽의 계산 결과를 붙인 형태이다.
  - 보이는 의미: 첫 번째 식의 계산 결과 제시.
- `84 ÷ 2 = 42`
  - 두 번째 나눗셈 식의 계산 결과를 붙인 형태이다.
  - 보이는 의미: 두 번째 식의 계산 결과 제시.
- `32<42`
  - 두 계산 결과의 크기를 비교하는 부등호 표현이다.
  - 보이는 의미: 첫 번째 값이 두 번째 값보다 작음을 표시.
- `따라서 32<42이므로 ...`
  - 비교식을 이유로 삼아 결론 문장으로 이어지는 구조이다.
  - 보이는 의미: 비교 결과를 바탕으로 선택 대상을 말함.

[빈칸 정책]
- 원본 화면에는 빈칸이 보이지 않는다.
- 따라서 빈칸을 새로 만들거나, 정답을 추론하여 채워 넣지 않는다.
- 만약 다른 버전에서 빈칸이 나타나더라도, 본 초안에서는 관찰된 그대로 빈칸을 유지해야 한다.
- `(정답)` 옆의 식도 화면에 보이는 요소로만 기록하고, 그것이 실제 정답이라는 의미를 추가로 해석하지 않는다.

[반복 패턴]
- 같은 형태의 나눗셈 식이 두 번 반복된다.
- 비교 박스 안에서는 식 두 개가 나란히 제시된다.
- 해설에서는 같은 두 식이 다시 나오며, 이번에는 계산 결과가 붙는다.
- 문장 구조도 반복된다:
  1. 비교 대상 제시
  2. 계산 결과 제시
  3. 비교식 제시
  4. 결론 문장 제시
- 좌우 2개 항목의 배치는 비교 문제임을 시각적으로 강조하는 반복 패턴이다.

[DSL 작성 힌트]
- 좌측 상단에 문제 번호와 지시문을 한 줄로 배치한다.
- 작은 네모 기호는 체크박스 또는 장식 요소처럼 보이므로, 별도 기호로 표현할지 여부를 TODO로 남겨도 좋다.
- 우측 상단의 분홍색 직사각형 박스는 얇은 테두리로 표현한다.
- 박스 내부의 두 식은 좌우로 나란히 두고, 서로 충분한 간격을 둔다.
- 박스 내부는 수평 정렬이 핵심이며, 글자 크기는 본문보다 약간 크게 보인다.
- 박스 아래 왼쪽에 `(정답)` 라벨과 식을 같은 줄에 둔다.
- 그 아래에 `(해설)` 라벨과 `96 ÷ 3 = 32, 84 ÷ 2 = 42`를 같은 줄에 둔다.
- 다음 줄에 `따라서 32<42이므로 묶이 더 작은 것은 96 ÷ 3입니다.`를 배치한다.
- 쉼표는 해설에서 두 결과를 구분하는 구분자 역할을 한다.
- `÷`, `=`, `<` 기호는 화면에 보이는 그대로 유지한다.
- 문단 사이 세로 간격을 충분히 두어, 박스-정답-해설의 위계가 드러나게 한다.

[semantic/solvable 힌트]
- 보이는 정보만 보면, 이 문제는 두 나눗셈 식을 비교하는 형식이다.
- 해설에는 각 식의 결과와 비교식이 이미 인쇄되어 있다.
- 다만 이 초안에서는 계산의 옳고 그름을 검증하지 않는다.
- 정답을 새로 계산하거나 보정하지 않고, 화면에 인쇄된 수식 관계만 기록한다.
- `나누기 n은 분모의 ×n으로 반영된다.`  
  - 이 문장은 원본에 직접 나타난 변환식이 아니라, 현재 화면에서는 참고용으로만 둘 수 있는 일반 규칙이다.
  - 다만 본 화면에는 분수 변환 체인이 보이지 않으므로, 실제 DSL에는 해당 규칙을 억지로 연결하지 않는다.

[주의 및 TODO]
- `묶이` 표기는 원문 그대로 유지한다. 오타인지 판독 오류인지 단정하지 않는다.
- `(정답)`에 적힌 `96 ÷ 3`은 화면에 보이는 사실로만 취급한다.
- 해설의 계산 결과 `32`, `42`가 맞는지 여부는 판정하지 않는다.
- 분홍색 박스의 정확한 좌표, 크기, 테두리 두께는 TODO로 남길 수 있다.
- 왼쪽 위의 작은 네모 기호가 체크박스인지 장식인지 확정하지 않는다.
- 정답을 빈칸에 채워 넣는 방식은 금지한다.
- 본 화면은 빈칸형 문제가 아니므로, 빈칸 관련 DSL 요소는 불필요할 수 있다.
