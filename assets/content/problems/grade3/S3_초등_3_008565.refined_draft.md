# [문제 개요]
- 문제 번호는 `30.`으로 보인다.
- 지시문은 `계산 결과가 2000보다 큰 곱셈식을 선택해 보세요.`이다.
- 화면 상단에는 구름 모양 테두리 안에 곱셈식 4개가 배치되어 있다.
- 왼쪽 아래에는 해설처럼 보이는 비교식 4줄이 세로로 정렬되어 있다.
- 가장 아래 왼쪽에는 작은 구름 모양 요소가 하나 더 보이며, 안에 `36 × 60`이 들어 있다.

# [원본에 보이는 요소]
## 보이는 사실
- 상단 텍스트:
  - `30.`
  - `계산 결과가 2000보다 큰 곱셈식을 선택해 보세요.`
- 구름 안 수식:
  - `19 × 90`
  - `20 × 70`
  - `40 × 50`
  - `36 × 60`
- 해설/비교 줄:
  - `(해설)19 × 90 = 1710 < 2000`
  - `20 × 70 = 1400 < 2000`
  - `40 × 50 = 2000 = 2000`
  - `36 × 60 = 2160 > 2000`
- 하단 작은 구름:
  - `36 × 60`

## 보이는 배치/형태
- 구름형 테두리는 분홍색 윤곽선으로 보인다.
- 각 구름 안에는 곱셈식이 1개씩 중앙 정렬에 가깝게 들어 있다.
- 해설은 왼쪽 아래에 세로 목록처럼 배치되어 있다.
- 해설 각 줄은 `곱셈식 → 계산 결과 → 비교기호 → 2000` 흐름으로 읽힌다.
- 비교기호는 원 안에 들어간 것처럼 보이는 `<`, `=`, `>` 표시가 쓰였다.
- 하단 왼쪽의 작은 구름은 화면 가장자리와 겹치며 일부만 보이는 듯한 배치이다.

## 추론되는 수학적 의미
- 각 곱셈식의 계산 결과를 2000과 비교하는 문제로 보인다.
- 해설 영역은 각 식의 계산 결과와 2000의 대소 관계를 보여 주는 설명 영역으로 해석된다.
- 다만 화면만으로는 실제 정답 표시나 선택 결과가 따로 강조되었는지는 확정할 수 없다.

# [수식/관계 구조]
## 보이는 구조
- 상단 곱셈식 4개는 서로 독립된 항목이다.
- 해설 각 줄은 다음과 같은 구조를 가진다.
  - `곱셈식 = 계산값 < 2000`
  - `곱셈식 = 계산값 = 2000`
  - `곱셈식 = 계산값 > 2000`

## 등호/비교 기호의 의미
- 첫 번째 `=`는 곱셈의 계산 결과를 적는 표시로 보인다.
- 뒤의 `<`, `=`, `>`는 계산 결과와 `2000`을 비교한 표시로 보인다.
- 따라서 각 줄은 “곱셈 결과를 구한 뒤 2000과 비교”하는 한 줄 설명이다.

## 등호별 변환 의미
- `19 × 90 = 1710 < 2000`
  - `19 × 90`의 계산 결과를 `1710`으로 적고, 그 값이 `2000`보다 작다는 뜻으로 보인다.
- `20 × 70 = 1400 < 2000`
  - 계산 결과 `1400`이 `2000`보다 작다는 뜻으로 보인다.
- `40 × 50 = 2000 = 2000`
  - 계산 결과가 `2000`이며, `2000`과 같다는 뜻으로 보인다.
- `36 × 60 = 2160 > 2000`
  - 계산 결과 `2160`이 `2000`보다 크다는 뜻으로 보인다.

## 변환 체인 관련 주의
- 별도의 연쇄적인 식 변형 과정은 보이지 않는다.
- 일반적인 등식 변형 풀이가 아니라, 계산 결과와 비교를 나란히 적은 형태이다.

# [빈칸 정책]
- 화면에 보이는 범위에는 빈칸이 확인되지 않는다.
- 따라서 빈칸을 추론해 채우지 않는다.
- 만약 원본 전체판에 별도의 선택칸이나 정답칸이 있다 해도, 현재 보이는 이미지 기준으로는 임의로 생성하지 않는다.

# [반복 패턴]
- 구름 모양 박스 안에 곱셈식을 넣는 형식이 반복된다.
- 해설 줄도 같은 문법으로 반복된다:
  - `곱셈식 = 계산값 비교기호 2000`
- `×` 기호가 여러 번 반복되어 등장한다.
- `2000`이 모든 비교의 기준값으로 반복 사용된다.
- 하단의 작은 구름은 상단 구름과 같은 스타일을 재사용한 반복 요소로 보인다.

# [DSL 작성 힌트]
- 상단 영역은 `문제 번호 + 지시문`을 한 줄 텍스트로 두는 구성이 적합하다.
- 상단의 4개 곱셈식은 각각 독립된 구름형 도형으로 배치하면 된다.
- 구름 도형은 직사각형이 아니라 둥근 물결형 외곽선으로 처리하는 것이 중요하다.
- 각 구름 내부 텍스트는 중앙 정렬에 가깝게 배치한다.
- 해설 영역은 왼쪽 아래 세로 목록으로 배치한다.
- 해설 각 줄은 다음의 3단 또는 4단 시각 구조로 구현하면 유사하다.
  - 왼쪽: 곱셈식
  - 중간: 계산 결과
  - 그 다음: 비교기호
  - 오른쪽: `2000`
- 비교기호는 일반 산술 등호가 아니라 비교 심볼로 별도 렌더링하는 것이 좋다.
- 하단의 작은 구름은 화면 아래쪽과 왼쪽 경계에 걸쳐 일부만 보이도록 배치하는 것이 원본과 가깝다.
- 선택 표시용 체크박스나 강조 테두리는 보이지 않으므로, 현재 이미지 기준으로는 추가하지 않는 편이 안전하다.

# [semantic/solvable 힌트]
- 문제의 의미는 “주어진 곱셈식들 중 계산 결과가 2000보다 큰 것을 고르기”로 보인다.
- 해설 영역은 각 식이 2000보다 작은지, 같은지, 큰지를 보여 주는 참고 설명이다.
- 화면에 보이는 텍스트만 기준으로는 `36 × 60`이 `2000`보다 큰 것으로 적혀 있다.
- 그러나 정답 선택을 최종적으로 단정하는 표현은 원본의 선택 표시가 확인되기 전까지는 피해야 한다.
