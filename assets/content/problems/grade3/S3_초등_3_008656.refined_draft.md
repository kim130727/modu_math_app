[문제 개요]
- 문제 번호는 `31.`로 보인다.
- 문제 문장은 `원의 지름이 아닌 것을 찾아 기호를 모두 선택하세요.`이다.
- 오른쪽에는 원 도형이 크게 배치되어 있고, 원 둘레 주변에 여러 기호 라벨이 붙어 있다.
- 아래쪽에는 `(정답)`과 `(해설)`이 따로 인쇄되어 있다.
- 원본 이미지 기준으로 하단 정답 표기와 해설 문장이 실제로 보인다.

[원본에 보이는 요소]
- 왼쪽 상단:
  - 체크박스처럼 보이는 작은 네모 기호
  - `31.`
  - 문제 문장 `원의 지름이 아닌 것을 찾아 기호를 모두 선택하세요.`
- 오른쪽 도형:
  - 원 1개
  - 원의 중심으로 보이는 작은 점 1개
  - 원 내부를 가로지르는 여러 선분/직선들
  - 원 둘레 주변의 원형 라벨 7개
- 원 둘레 라벨에 보이는 글자:
  - `ㄴ`
  - `ㄷ`
  - `ㄹ`
  - `ㅁ`
  - `ㅂ`
  - `ㅅ`
  - `ㄱ`
- 하단:
  - `(정답) ㄷ, ㅁ`
  - `(해설) 중심을 지나지 않는 ㄷ, ㅁ이 원의 지름이 아닙니다.`

[수식/관계 구조]
- 등호로 이어진 계산식이나 변형식은 보이지 않는다.
- 수학적 관계는 식이 아니라 도형의 위치 관계로 제시된다.
- 핵심 관계는 다음과 같이 읽힌다.
  - 어떤 선이 원의 중심을 지나면 지름으로 볼 수 있는 구조
  - 중심을 지나지 않는 선은 지름이 아님
- 원 내부의 여러 선은 서로 다른 방향으로 배치되어 있어, 각 선이 중심점을 통과하는지 여부를 구분하는 문제 구조이다.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄 변형이 보이지 않는다.
- 계산 변형 설명이 필요한 식 구조도 보이지 않는다.

[빈칸 정책]
- 이 문제는 빈칸을 직접 채우는 형식이 아니라, 기호를 선택하는 형식으로 보인다.
- 이미지에 `(정답)`이 인쇄되어 보이지만, 정답값은 추론하지 않고 **보이는 그대로**만 기록한다.
- DSL 작성 시에도 보이지 않는 값은 임의로 채우지 말고, 라벨/정답 표시는 빈 상태 또는 텍스트 슬롯으로 유지한다.
- 특히 해설이나 정답 박스가 있어도, 그 내용을 근거로 새로운 값을 생성하지 않는다.

[반복 패턴]
- 원 둘레 바깥에 작은 원형 라벨이 여러 개 반복된다.
- 각 라벨 안에는 한글 자모 1개씩 들어간다.
- 원 내부에는 선분들이 여러 개 반복적으로 교차하며, 중심점을 기준으로 대칭처럼 보이는 요소도 있다.
- 전체적으로
  - `원`
  - `중심점`
  - `여러 선분`
  - `주변 기호 라벨`
  의 반복 구조로 구성된다.
- 라벨은 선분 끝점과 직접 연결된 표시라기보다, 각각의 선분을 구분하는 선택 기호처럼 배치되어 있다.

[DSL 작성 힌트]
- 상단 왼쪽에 문제 번호와 문장을 한 줄 텍스트로 배치한다.
- 오른쪽에는 큰 원을 그리고, 중앙에 작은 점을 찍어 중심을 표시한다.
- 원 내부에는 여러 직선을 서로 다른 방향으로 배치해 교차 구조를 표현한다.
- 원 둘레 주변에는 작은 원형 라벨 7개를 배치하고, 각 라벨에 `ㄴ, ㄷ, ㄹ, ㅁ, ㅂ, ㅅ, ㄱ`을 넣는다.
- 하단에는 `(정답)`과 `(해설)`을 별도 텍스트 줄로 둔다.
- 렌더링은 좌우 분리형 구성이 적합하다:
  - 왼쪽: 문제 문장과 하단 문구
  - 오른쪽: 도형 그림
- 라벨은 원 바깥쪽에 조금 띄워 놓아야 하며, 선분과 겹치지 않도록 주의한다.
- 원 중심점은 아주 작은 점으로 표시하고, 다른 교차점과 구분되게 한다.

[semantic/solvable 힌트]
- 문제의 의미는 “원의 지름인지 아닌지”를 도형에서 구분하는 것이다.
- 해설에 따르면, `중심을 지나지 않는 ㄷ, ㅁ`이 지름이 아니라고 적혀 있다.
- 다만 이 문서는 풀이를 새로 만들지 않으며, 원본에 인쇄된 내용을 관찰 사실로만 옮긴다.
- 따라서 DSL에서는
  - 기호 선택형 문제임
  - 도형의 중심 통과 여부가 핵심 판단 기준임
  을 나타내면 충분하다.

[주의 및 TODO]
- 정답을 추론해서 다른 기호를 추가로 판단하지 말 것.
- `(정답) ㄷ, ㅁ`은 원본에 보이지만, 최종 정답으로 단정하는 서술은 피하고 관찰 사실로 취급할 것.
- 원 내부 선분의 정확한 끝점과 교차 관계는 해상도상 약간 애매할 수 있으므로, 그림 재현 시 중심 통과 여부가 드러나도록만 구성한다.
- 라벨 `ㄱ`과 `ㄴ` 등의 정확한 위치 관계는 원본을 다시 확인해 필요한 경우 보정한다.
- 원본에서 보이는 선들 중 일부는 서로 겹쳐 보이므로, 각 선을 독립 요소로 그릴지 여부는 TODO로 남긴다.
- 해설 문장의 핵심은 `중심을 지나지 않는`이라는 조건이므로, 이 문구가 깨지지 않도록 텍스트 배치를 유지한다.
