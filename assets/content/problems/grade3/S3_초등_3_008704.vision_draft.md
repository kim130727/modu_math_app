# [전체적인 구성]
- 한 장의 초등 수학 문제 화면이다.
- 왼쪽에는 **파란 점선 격자 위에 원 3개**가 겹쳐 그려진 그림이 있고, 중심을 나타내는 **작은 분홍색 점**이 보인다.
- 오른쪽에는 말풍선 속 설명을 하는 **캐릭터(승아)**가 있다.
- 아래쪽에는 **연한 초록색 배경의 선택 문장 영역**이 있으며, 괄호로 된 선택지와 수치가 들어간 문장이 보인다.
- 화면 맨 아래에는 **정답 표시**와 **해설 문장**이 텍스트로 배치되어 있다.

# [보이는 텍스트]
- 상단 문제 번호와 문장:
  - `□ 98. 승아는 규칙에 따라 원을 그렸습니다. 승아가 원을 그린 규칙을 찾아 알맞은 것을 선택해보세요.`
- 말풍선 안:
  - `원의 중심이 같고`
  - `원의 반지름은 다르게 하여`
  - `그린 거야.`
- 캐릭터 이름표:
  - `승아`
- 연한 초록색 박스 안:
  - `원의 중심이 ( 같습니다, 다릅니다 ).`
  - `원의 반지름이 모두 ( 1, 2, 3 )칸씩 늘어났습니다.`
- 하단 정답 표기:
  - `(정답) 같습니다, 1`
- 하단 해설:
  - `(해설) 원의 중심이 같습니다.`
  - `원의 반지름이 모두 1칸씩 늘어났습니다.`

# [반복되는 시각 패턴]
- 왼쪽 그림에서는 **동심원 형태의 원 3개**가 반복되어 보인다.
- 각 원은 같은 중심을 공유하는 것처럼 배치되어 있으며, 바깥쪽으로 갈수록 반지름이 커진다.
- 배경의 파란 점선 격자는 **정사각형 칸들이 규칙적으로 반복**된 형태이다.
- 초록색 선택 영역 안에는 같은 문장 구조에서 **괄호 안 선택지**가 반복되는 형식이 있다:
  - `가( 같습니다, 다릅니다 ).`
  - `모두 ( 1, 2, 3 )칸씩 ...`
- 원 그림의 왼쪽과 초록 박스의 문장이 서로 대응되는 문제 구성이다.

# [수식 단계]
- 이 문제 화면에는 **등호를 포함한 수식 변형 단계가 보이지 않는다.**
- 따라서 식의 전개나 변형 과정은 관찰되지 않는다.

# [등호별 변환 의미]
- 해당 없음.
- 화면에 수식의 등호 연결이 나타나지 않으므로, 단계별 변환 의미를 설명할 대상이 없다.

# [세부 내용]
- **왼쪽 도형**
  - 파란 점선 격자 위에 원이 3개 겹쳐 있다.
  - 가장 안쪽 원, 가운데 원, 가장 바깥 원이 있으며 모두 거의 같은 중심에 맞춰 배치된 것으로 보인다.
  - 중심 위치에는 **작은 분홍색 점**이 표시되어 있다.
  - 원들은 서로 다른 크기이며, 중심에서 바깥으로 갈수록 커진다.
- **오른쪽 말풍선**
  - 캐릭터 승아가 손가락을 들어 설명하는 모습이다.
  - 말풍선은 타원형이며 아래쪽에 꼬리가 있어 캐릭터를 가리킨다.
  - 말풍선 텍스트는 줄바꿈되어 3줄로 배치되어 있다.
- **중앙 초록색 박스**
  - 연한 초록색 직사각형 배경 안에 두 개의 문장이 세로로 배치되어 있다.
  - 첫 문장은 중심에 대한 선택 문장이고, 두 번째 문장은 반지름 증가량에 대한 선택 문장이다.
  - 각 문장 안의 선택지는 괄호로 묶여 있으며, 쉼표로 구분된 단어/숫자들이 나열되어 있다.
- **하단**
  - 정답 표기는 `(정답)` 뒤에 선택된 답이 적혀 있다.
  - 해설 표기는 `(해설)` 뒤에 짧은 설명 문장이 이어진다.
  - 해설의 첫 줄과 둘째 줄은 왼쪽 정렬로 보이며, 둘째 줄은 약간 들여쓴 느낌으로 배치되어 있다.

# [Python DSL 작성 힌트]
- 전체 화면을 크게 다음 영역으로 나누어 기술하면 좋다:
  1. 상단 문제 문장 영역
  2. 왼쪽의 격자+원 그림 영역
  3. 오른쪽 말풍선+캐릭터 영역
  4. 중앙 하단의 연두색 선택문 영역
  5. 맨 아래 정답/해설 텍스트 영역
- 왼쪽 그림은 다음 요소를 따로 분리해서 그리면 좋다:
  - 점선 격자 배경
  - 동심원 3개
  - 중심점 표시
- 말풍선은 **타원형 테두리와 아래 꼬리**를 가진 도형으로 표현할 수 있다.
- 선택 문장 영역은 **연한 초록색 직사각형 박스**로 처리하고, 내부 텍스트는 두 줄로 배치하면 된다.
- 괄호 안 선택지는 **선택지 리스트처럼 보이도록** 텍스트로 유지해야 하며, 답을 미리 강조하지 않는 편이 좋다.
- 하단의 `(정답)`, `(해설)`은 문제 이미지의 일부로서 별도 텍스트 블록으로 넣을 수 있다.

# [주의할 점]
- **정답을 추론해서 빈칸에 넣지 마세요.**
- 그림만 보고 중심의 정확한 좌표나 반지름의 정확한 수치를 단정하지 마세요.
- 원들이 정확히 몇 칸씩 넓어졌는지는 화면의 해설 텍스트에 보이는 값만 그대로 다뤄야 하며, 임의로 계산해 확정하지 마세요.
- 선택지 `( 같습니다, 다릅니다 )`, `( 1, 2, 3 )`는 **보이는 그대로** 보존해야 한다.
- 말풍선의 문장은 의미상 해석이 가능하더라도, 문장 자체는 그대로 옮기는 것이 좋다.
- 원 중심을 나타내는 점의 색이 분홍색으로 보이지만, 기기/압축에 따라 약간 다르게 보일 수 있다.

# [불확실한 부분]
- 왼쪽 원 그림에서 **각 원의 정확한 반지름 길이**는 이미지 해상도상 명확히 수치화하기 어렵다.
- 파란 격자의 칸 수를 정확히 셀 수는 있으나, 문제 작성용 DSL에서는 굳이 수치를 확정하지 않아도 된다.
- 캐릭터의 세부 얼굴 표정과 의상 문양은 작고 일부 흐려 보여, 세부 묘사는 생략하거나 간단히 처리하는 것이 안전하다.
- 상단 문장 맨 앞의 작은 네모 표시는 문제 번호 표기용인지 도형인지 약간 애매하므로, 필요하면 단순한 기호로만 취급하는 것이 좋다.
